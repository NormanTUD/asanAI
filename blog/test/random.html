Understanding Pseudo-Random Number Generators — from simple to cryptographically secure

What is a PRNG?

A Pseudo-Random Number Generator (PRNG) is a deterministic algorithm that produces a sequence of numbers that appears random but is entirely determined by an initial value called the seed.

1. Linear Congruential Generator (LCG)

⚠️ Not Cryptographically Secure

The LCG is one of the oldest and simplest PRNGs. It was introduced by Lehmer in 1951. Many early standard library rand() functions used LCGs.

The Formula

Where:

  • a = multiplier
  • c = increment
  • m = modulus (determines the period)
  • sn = current state

Implementation

class LCG { constructor(seed, a = 1664525, c = 1013904223, m = 2**32) { this.state = seed; this.a = a; this.c = c; this.m = m; } next() { // s_{n+1} = (a * s_n + c) mod m this.state = (this.a * this.state + this.c) % this.m; return this.state / this.m; // normalize to [0, 1) } }

Step-by-Step Execution

⚠️ Why it's insecure: Given just a few outputs, an attacker can solve for the parameters (a, c, m) and predict all future values. The state space is small and the relationship is linear — trivially invertible.

2. Xorshift Generator

⚠️ Not Cryptographically Secure

Xorshift was proposed by George Marsaglia in 2003. It uses only XOR and bit-shift operations, making it extremely fast. It's used in many game engines and simulations.

The Algorithm (xorshift32)

The key insight: XOR with shifted versions of itself creates enough "mixing" for statistical randomness, but not enough for cryptographic security.

Implementation

class Xorshift32 { constructor(seed) { this.state = seed || 1; // must not be 0 } next() { let x = this.state; x ^= x << 13; // shift left, XOR x ^= x >>> 17; // shift right (unsigned), XOR x ^= x << 5; // shift left, XOR this.state = x >>> 0; // ensure unsigned 32-bit return this.state / 0xFFFFFFFF; } }

Visualizing the Bit Manipulation

⚠️ Why it's insecure: The entire state IS the output. An attacker who sees one output knows the full internal state and can predict every future (and past) value. The operations are all linear over GF(2).

3. xoshiro128** (V8/Chrome's Math.random)

⚠️ Not Cryptographically Secure

Modern browsers like Chrome (V8 engine) use xoshiro128** for Math.random(). It has a 128-bit state and excellent statistical properties, but is still predictable.

Implementation (simplified 64-bit version concept)

// Conceptual xoshiro128** (actual uses 64-bit math) class Xoshiro128ss { constructor(s0, s1, s2, s3) { this.s = [s0, s1, s2, s3]; // 128-bit state } rotl(x, k) { return ((x << k) | (x >>> (32 - k))) >>> 0; } next() { const s = this.s; // Output function: scramble s[1] * 5, rotate, * 9 const result = (this.rotl((s[1] * 5) >>> 0, 7) * 9) >>> 0; // State update const t = (s[1] << 9) >>> 0; s[2] ^= s[0]; s[3] ^= s[1]; s[1] ^= s[2]; s[0] ^= s[3]; s[2] ^= t; s[3] = this.rotl(s[3], 11); return result / 0xFFFFFFFF; } }
⚠️ Why it's insecure: With ~5 outputs from Math.random(), an attacker can use a constraint solver (like Z3) to recover the full 128-bit state and predict all future outputs. The transformations, while non-linear in the output function, are still solvable.

4. Blum Blum Shub (BBS)

✓ Cryptographically Secure (theoretically)

The Blum Blum Shub generator (1986) is one of the simplest CSPRNGs. Its security is based on the difficulty of factoring large integers — the same hardness assumption behind RSA.

The Formula

Where:

  • M = p × q where p and q are large primes, both ≡ 3 (mod 4)
  • The output is just the least significant bit (or few bits) of each state
  • Security relies on the Quadratic Residuosity Problem

Implementation (small demo values)

class BlumBlumShub { constructor(seed, p = 499, q = 547) { // p and q must be primes ≡ 3 (mod 4) this.M = p * q; // 499 * 547 = 272953 this.state = (seed * seed) % this.M; // x₀ = seed² mod M } next() { // x_{n+1} = x_n² mod M this.state = (this.state * this.state) % this.M; return this.state & 1; // output = least significant bit } nextByte() { let byte = 0; for (let i = 0; i < 8; i++) { byte = (byte << 1) | this.next(); } return byte; } }

Step-by-Step Execution

✓ Why it's secure: To predict the next bit, an attacker would need to compute square roots modulo M, which requires knowing the factorization of M (p and q). With large primes (1024+ bits each), this is computationally infeasible. However, BBS is very slow compared to modern CSPRNGs.

5. Counter-Mode DRBG (CTR_DRBG) — How crypto.getRandomValues works

✓ Cryptographically Secure

Modern operating systems and browsers use CTR_DRBG (NIST SP 800-90A) or similar constructions. The Web Crypto API's crypto.getRandomValues() uses the OS's CSPRNG which typically employs AES in counter mode.

The Concept

(hardware, interrupts, etc.)
→
Key Derivation
→
AES-CTR

→
Random Output
(128 bits per block)

Simplified Implementation Concept

class CTR_DRBG_Concept { constructor(entropy) { // In reality, uses hardware entropy sources this.key = deriveKey(entropy); this.counter = 0; } generate(numBytes) { const blocks = Math.ceil(numBytes / 16); let output = new Uint8Array(numBytes); let offset = 0; for (let i = 0; i < blocks; i++) { // Each block = AES_Encrypt(key, counter) const block = AES_Encrypt(this.key, this.counter); this.counter++; // Copy block to output const copyLen = Math.min(16, numBytes - offset); output.set(block.slice(0, copyLen), offset); offset += copyLen; } // Periodically reseed with fresh entropy this.reseed(); return output; } } // In the browser, you simply use: const array = new Uint32Array(10); crypto.getRandomValues(array); // Uses OS CSPRNG

Live Demo: crypto.getRandomValues()

✓ Why it's secure: Breaking this requires breaking AES (no known practical attack). The entropy comes from physical sources (CPU jitter, interrupt timing, hardware RNG). Even if you see all outputs, you cannot determine the key or predict future outputs without breaking AES-256.

Rule of Thumb: If the word "security," "token," "key," "password," "nonce," or "secret" appears anywhere near your use case, use crypto.getRandomValues() (browser) or crypto.randomBytes() (Node.js). For everything else, Math.random() is fine.
