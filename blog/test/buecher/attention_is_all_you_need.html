🧠 Attention Is All You Need

Interaktive Erklärung des Attention-Mechanismus
Basierend auf Vaswani et al., 2017 (arXiv:1706.03762)

🗺️Überblick: Wo steht Attention?

Du verstehst bereits:

  • Embeddings – Wörter als Punkte in einem hochdimensionalen Raum (dmodel = 512), wo Richtungen Bedeutungen kodieren
  • FFNs – Können beliebige Funktionen durch Stückelung approximieren (Universal Approximation)

Attention ist das fehlende Puzzlestück: Es ermöglicht jedem Token, Informationen von allen anderen Tokens zu sammeln – gewichtet nach Relevanz. Ohne Attention wäre jedes Token isoliert in seinem eigenen Embedding gefangen.

Die Kernidee in einem Satz:

"Attention berechnet für jedes Wort eine gewichtete Mischung aller anderen Wörter –
wobei die Gewichte durch Ähnlichkeit (Dot-Product) bestimmt werden."

🔑Schritt 1: Query, Key, Value

Jedes Token-Embedding wird durch drei verschiedene lineare Projektionen in drei Vektoren transformiert:

🔍 Query (Q) – "Was suche ich?"

Der Query-Vektor repräsentiert, welche Art von Information dieses Token von anderen Tokens braucht.

🗝️ Key (K) – "Was biete ich an?"

Der Key-Vektor repräsentiert, welche Art von Information dieses Token anderen Tokens anbieten kann.

💎 Value (V) – "Was ist mein Inhalt?"

Der Value-Vektor enthält die tatsächliche Information, die weitergegeben wird, wenn ein anderes Token "aufmerksam" auf dieses Token ist.

Q = X · WQ   |   K = X · WK   |   V = X · WV

WQ, WK ∈ ℝd_model × d_k, WV ∈ ℝd_model × d_v sind gelernte Gewichtsmatrizen.

⚡Schritt 2: Scaled Dot-Product Attention

Die zentrale Formel des Papers:

Attention(Q, K, V) = softmax(Q·KT / √dk) · V

Das passiert in 4 Teilschritten:

2
3

Schritt 1: Dot-Product (Q · KT)

Berechne die Ähnlichkeit zwischen jedem Query und jedem Key durch das Skalarprodukt. Hoher Wert = hohe Ähnlichkeit = "dieses Token ist relevant für mich".

Jede Zelle zeigt: Wie sehr "passt" der Query von Token i zum Key von Token j?

Schritt 2: Skalierung (÷ √dk)

Warum skalieren? Wenn dk groß ist, werden die Dot-Products sehr groß. Große Werte drücken den Softmax in Regionen mit extrem kleinen Gradienten → schlechtes Lernen.

64

Beobachte: Je größer d_k, desto extremer die Werte VOR der Skalierung. Die Skalierung normalisiert das.

Schritt 3: Softmax

Softmax wandelt die skalierten Scores in eine Wahrscheinlichkeitsverteilung um (alle Werte zwischen 0 und 1, Summe = 1). Jede Zeile sagt: "Wie viel Aufmerksamkeit schenke ich jedem anderen Token?"

Mask aus: Encoder Self-Attention (jedes Token sieht alle)

Schritt 4: Gewichtete Summe der Values

Die Attention-Gewichte (aus Softmax) werden mit den Value-Vektoren multipliziert. Das Ergebnis: Jedes Token bekommt eine gewichtete Mischung der Informationen aller anderen Tokens.

👁️‍🗨️Schritt 3: Multi-Head Attention

Statt EINE Attention mit dmodel=512 Dimensionen zu berechnen, teilt das Paper die Dimensionen auf h=8 Köpfe auf (je dk=64).

MultiHead(Q,K,V) = Concat(head₁, ..., headh) · WO
wobei headi = Attention(Q·WiQ, K·WiK, V·WiV)

Warum? Jeder Kopf kann auf verschiedene Aspekte achten:

  • Kopf 1: Syntaktische Beziehungen (Subjekt → Verb)
  • Kopf 2: Anaphora-Auflösung ("its" → "The Law")
  • Kopf 3: Lokale Nachbarschaft
  • Kopf 4: Fernabhängigkeiten ("making...more difficult")
  • ...

📍Schritt 4: Positional Encoding

Attention ist positions-agnostisch – "Die Katze sitzt" und "sitzt Katze Die" wären identisch! Deshalb werden Positionsinformationen über Sinus/Cosinus-Funktionen hinzugefügt:

PE(pos, 2i) = sin(pos / 100002i/d_model)
PE(pos, 2i+1) = cos(pos / 100002i/d_model)

Jede Dimension hat eine andere Frequenz → das Modell kann relative Positionen als lineare Transformation lernen.

🔄Schritt 5: Alles zusammen – Ein Layer

Ein Encoder-Layer besteht aus:

  1. Multi-Head Self-Attention + Residual Connection + LayerNorm
  2. Position-wise FFN + Residual Connection + LayerNorm

Das wird N=6 mal gestapelt.

💡Warum Self-Attention statt RNN/CNN?

Self-Attention

  • Komplexität: O(n² · d)
  • Sequentielle Ops: O(1)
  • Max. Pfadlänge: O(1) ✨

Recurrent (RNN)

  • Komplexität: O(n · d²)
  • Sequentielle Ops: O(n) 🐌
  • Max. Pfadlänge: O(n)

Kernvorteil: Jedes Token kann direkt (Pfadlänge 1!) auf jedes andere Token zugreifen. Bei RNNs muss Information durch n Schritte fließen → Langzeitabhängigkeiten gehen verloren.
