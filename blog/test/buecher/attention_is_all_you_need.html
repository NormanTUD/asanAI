Attention – Geometrisch Verstanden

Basierend auf "Attention Is All You Need" (Vaswani et al., 2017) – visuell & mathematisch

1. Die Kernidee – Was tut Attention geometrisch?

Was du schon weißt: Embeddings sind Punkte in einem hochdimensionalen Raum. Jede Richtung kann eine Bedeutung tragen (männlich↔weiblich, konkret↔abstrakt, etc.). FFNs können beliebige Funktionen approximieren.

Was Attention hinzufügt: Attention erlaubt jedem Token, gezielt Information von anderen Tokens einzusammeln. Es ist ein inhaltsabhängiges Routing: "Wen frage ich, und was bekomme ich zurück?"

Geometrisch in einem Satz: Jedes Token schickt eine "Frage" (Query) in den Raum. Alle anderen Tokens haben "Schilder" (Keys). Je ähnlicher die Richtung von Query und Key, desto mehr "Information" (Value) fließt zurück. Das Ergebnis ist eine gewichtete Mischung der Values – ein neuer Punkt im Raum.

↑ 2D-Vereinfachung: "sitzt" (rot) fragt den Raum ab. Dicke Linien = hohe Attention. Der grüne Diamant ist der Output.

2. Unterräume herausschneiden – WQ, WK, WV

Der 512-dimensionale Embedding-Raum enthält ALLES über ein Wort: Syntax, Semantik, Position, Kontext. Attention braucht aber spezialisierte Sichten:

  • Query-Raum (64d): "Was suche ich?" – z.B. "Ich bin ein Verb und suche mein Subjekt"
  • Key-Raum (64d): "Was biete ich an?" – z.B. "Ich bin ein Nomen und könnte ein Subjekt sein"
  • Value-Raum (64d): "Was gebe ich weiter?" – z.B. "Meine semantische Information: belebt, weiblich, Tier"

Die Matrizen WQ, WK, WV schneiden schräge Unterräume aus dem vollen 512d-Raum heraus.

Geometrisch: Schräge Projektion

↑ Der volle Raum (grau) wird durch WQ auf einen schrägen Unterraum (rot) projiziert. Verschiedene Aspekte werden betont.

Warum schräg?

Stell dir den 512d-Raum vor. Darin gibt es Richtungen wie:

  • Achse 1–10: Syntax-Info (Wortart, Kasus...)
  • Achse 11–50: Semantik (Bedeutung)
  • Achse 51–80: Position im Satz
  • ... usw.

WQ könnte z.B. hauptsächlich die Syntax-Achsen extrahieren (um zu fragen: "Wer passt syntaktisch zu mir?"), während WV die Semantik-Achsen extrahiert (um zu antworten: "Hier ist meine Bedeutung").

In 2D/3D sieht das so aus: Der Unterraum liegt schräg im Oberraum – er ist keine achsenparallele Scheibe, sondern eine beliebig orientierte Ebene.

3. Q · K – Ähnlichkeit im Unterraum messen

Nachdem Query und Key in ihren gemeinsamen Unterraum projiziert wurden, misst das Skalarprodukt (Dot-Product), wie gut sie zueinander passen. Geometrisch: Es ist die projizierte Länge des einen Vektors auf den anderen.

↑ Drehe den Query (weiß). Das Dot-Product mit jedem Key ist die Projektion auf die Query-Richtung.

Intuition:

  • Query und Key zeigen in gleiche Richtung → großes positives Dot-Product → hohe Attention
  • Senkrecht (90°) → Dot-Product ≈ 0 → keine Attention
  • Entgegengesetzt → negativ → nach Softmax fast 0

In 64 Dimensionen:

Aber das Prinzip ist identisch – es misst die Ausrichtung der Vektoren.

Warum durch √dk teilen? In 64 Dimensionen hat das Dot-Product eine Varianz von ~64. Das drückt Softmax in Sättigung (Gradient ≈ 0). Division durch √64 = 8 normalisiert die Varianz auf ~1.

4. Softmax – Von Scores zu Gewichten

Softmax verwandelt die skalierten Scores in eine Wahrscheinlichkeitsverteilung: Alle Gewichte sind positiv und summieren zu 1. Geometrisch bestimmt es, wie viel Prozent der Information von jedem Token fließt.

↑ Balkendiagramm: Attention-Gewichte für Query "sitzt" auf alle Keys

Niedrige Schärfe (hohe Temperatur): Gleichmäßige Verteilung – alle Tokens gleich wichtig.

Hohe Schärfe (niedrige Temperatur): Ein Token dominiert – fast wie ein harter Lookup.

Im Paper: Die Schärfe wird durch √dk kontrolliert. Ohne Scaling wäre die effektive Temperatur zu niedrig (zu spitz) in hohen Dimensionen.

5. Gewichtete Summe – Der Output als Punkt im Raum

Der Output ist eine konvexe Kombination der Value-Vektoren. Geometrisch: Der Output-Punkt liegt innerhalb des Simplex (Dreieck/Tetraeder/...) der Values. Die Gewichte bestimmen, wo genau.

↑ Bewege die Gewichte! Der grüne Output-Punkt wandert im Dreieck der Values.

Schlüssel-Einsicht: Attention kann den Output-Punkt nur innerhalb der konvexen Hülle der Values platzieren. Es kann keine "neue" Information erfinden – es kann nur bestehende Information mischen. Die FFN-Schicht danach kann dann nichtlinear transformieren.

6. Multi-Head – Parallele Perspektiven auf den Raum

Statt EINEN 512d-Unterraum zu nutzen, schneidet das Modell 8 verschiedene 64d-Unterräume heraus – jeder mit eigenen WQ, WK, WV. Jeder Head kann auf andere Aspekte achten:

  • Head 1: "Wer ist mein syntaktisches Subjekt?"
  • Head 2: "Welches Wort ist semantisch ähnlich?"
  • Head 3: "Was steht direkt neben mir?"
  • Head 4: "Wo ist das Verb zu diesem Nomen?"
  • ... etc.

↑ 8 Heads schneiden verschiedene schräge Ebenen aus dem Gesamtraum. Jeder sieht andere Beziehungen.

Head 1 (Syntax): Projiziert in einen Unterraum, in dem syntaktische Rollen betont werden. "Die" attended stark zu "Katze" (Artikel→Nomen).

Concat + WO: Die 8 Outputs (je 64d) werden konkateniert (→ 512d) und durch eine letzte Matrix WO zurück in den Modellraum projiziert. So fließen alle Perspektiven zusammen.

7. Das Gesamtbild – Warum funktioniert das so gut?

Die geometrische Essenz von Attention in 4 Schritten:

  1. Unterräume herausschneiden: WQ, WK projizieren in einen gemeinsamen "Vergleichsraum". Nur die relevanten Aspekte werden verglichen.
  2. Ähnlichkeit messen: Im Vergleichsraum zeigt das Dot-Product, welche Tokens zueinander passen (gleiche Richtung = relevant).
  3. Gewichte normalisieren: Softmax macht daraus eine Verteilung – "wie viel Prozent von jedem?"
  4. Information mischen: Die Values (aus einem ANDEREN Unterraum!) werden gewichtet gemischt → neuer Punkt im Raum.

↑ Klicke auf ein Token um es als Query zu wählen. Beobachte den gesamten Attention-Fluss.

Warum ist das so mächtig?

1. Inhaltsabhängiges Routing: Anders als CNNs (feste Nachbarschaft) oder RNNs (sequentiell) kann Attention jedes Token mit jedem anderen verbinden – und die Stärke hängt vom Inhalt ab.

2. Parallele Perspektiven: 8 Heads = 8 verschiedene "Fragen" gleichzeitig. Ein Head findet das Subjekt, ein anderer den Kontext, ein dritter die Negation.

3. O(1) Pfadlänge: Information fließt in EINEM Schritt von jedem Token zu jedem anderen. Kein sequentielles Durchreichen wie bei RNNs.

4. Lernbare Unterräume: Die W-Matrizen werden gelernt. Das Modell entscheidet selbst, welche Aspekte es vergleichen will und welche Information es weitergibt.
