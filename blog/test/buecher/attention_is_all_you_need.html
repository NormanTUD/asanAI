1. Die Grundidee: Was macht Attention?

Du verstehst bereits: Embeddings sind Punkte in hochdimensionalen Räumen, und jede Richtung kann eine Bedeutung kodieren. FFNs können beliebige Funktionen approximieren.

Was Attention hinzufügt: Attention ist der Mechanismus, der es jedem Token erlaubt, Informationen von anderen Tokens zu sammeln – und zwar gewichtet nach Relevanz. Es ist eine dynamische, inhaltsabhängige Informationsrouting-Funktion.

Geometrisch: Jedes Token "fragt" (Query) den Raum ab, welche anderen Tokens (Keys) relevante Informationen (Values) haben. Die Antwort ist eine gewichtete Mischung der Values.

2. Query, Key, Value – Drei Projektionen eines Tokens

Jedes Token-Embedding wird durch drei verschiedene lineare Projektionen (Matrizen) in drei verschiedene Vektoren transformiert:

  • Query (Q): "Was suche ich?" – Die Frage, die dieses Token stellt
  • Key (K): "Was biete ich an?" – Das Label, unter dem dieses Token gefunden werden kann
  • Value (V): "Was gebe ich weiter?" – Die tatsächliche Information, die weitergegeben wird

3. Dot-Product: Ähnlichkeit messen

Der Kern von Attention: Wie ähnlich ist mein Query zu jedem Key? Das Dot-Product (Skalarprodukt) misst genau das – es ist die projizierte Länge eines Vektors auf einen anderen, also ein Maß für ihre Ausrichtung im Raum.

↑ Bewege den Slider um den Query-Vektor zu drehen und sieh wie sich das Dot-Product ändert

Geometrisch:

  • Gleiche Richtung → großes positives Dot-Product (hohe Attention)
  • Orthogonal (90°) → Dot-Product = 0 (keine Attention)
  • Entgegengesetzt → negatives Dot-Product (nach Softmax: fast 0)

4. Scaling: Warum durch √dk teilen?

Problem: In hohen Dimensionen werden Dot-Products sehr groß (Varianz = dk). Das drückt die Softmax in Sättigungsbereiche mit winzigen Gradienten.

Lösung: Teile durch um die Varianz auf 1 zu normalisieren.

6. Gewichtete Summe der Values: Das Ergebnis

Der finale Schritt: Die Attention-Gewichte (aus Softmax) werden verwendet, um eine gewichtete Summe der Value-Vektoren zu berechnen. Geometrisch: Der Output-Vektor ist ein Punkt im Value-Raum, der zwischen den Values der relevanten Tokens liegt.

↑ Interaktiv: Ändere die Gewichte und sieh wie sich der Output-Vektor bewegt!

Geometrisch in 2D/3D (abstrahierbar auf 64D): Der Output ist ein Punkt, der im "Dreieck" (bzw. Simplex) der Value-Vektoren liegt. Die Gewichte bestimmen, wie nah er an welchem Value ist. Bei Gewicht 1.0 für einen Value liegt er genau auf diesem Value. Bei gleichmäßigen Gewichten liegt er im Schwerpunkt.

7. Multi-Head Attention: Parallele Perspektiven

Statt einer einzigen Attention-Berechnung führt das Modell h = 8 parallele Attention-Berechnungen durch, jede mit eigenen Projektionsmatrizen. Jeder "Head" kann auf andere Aspekte achten (z.B. syntaktische Beziehungen, semantische Ähnlichkeit, Positionsnähe).

Zusammenfassung der Formel

Was du jetzt verstehen solltest:

  1. Lineare Projektionen (WQ, WK, WV) transformieren Embeddings in Query/Key/Value-Räume
  2. Dot-Product misst die Ausrichtung (Ähnlichkeit) zwischen Query und Keys im Raum
  3. Scaling (÷√dk) verhindert, dass hohe Dimensionen die Softmax sättigen
  4. Softmax normalisiert zu einer Wahrscheinlichkeitsverteilung
  5. Gewichtete Summe der Values ergibt den Output – geometrisch ein Punkt im Konvex der Values
  6. Multi-Head erlaubt parallele, verschiedene Attention-Muster in Unterräumen
