Python-Implementierung

import numpy as np

def gradient_descent(f, grad_f, x0, lr=0.01, n_iter=1000, tol=1e-8):
    """Einfacher Gradient Descent."""
    x = np.array(x0, dtype=float)
    history = [x.copy()]
    
    for i in range(n_iter):
        g = grad_f(x)
        if np.linalg.norm(g) < tol:
            break
        x = x - lr * g
        history.append(x.copy())
    
    return x, history

# Beispiel: Rosenbrock
def rosenbrock(x):
    return (1 - x[0])**2 + 100*(x[1] - x[0]**2)**2

def grad_rosenbrock(x):
    dx = -2*(1-x[0]) - 400*x[0]*(x[1]-x[0]**2)
    dy = 200*(x[1] - x[0]**2)
    return np.array([dx, dy])

x_opt, hist = gradient_descent(rosenbrock, grad_rosenbrock, 
                                [-1.0, 1.0], lr=0.001)

Ax vs HyperOpt: Ax nutzt GPs (besser bei wenigen Evaluierungen, ~10-100). HyperOpt nutzt TPE (schneller, skaliert besser auf viele Dimensionen). Für HPC mit vielen parallelen Jobs ist Ax mit qEI/qNEI oft besser.

Warum Bayesian Optimization bei wenigen Evaluierungen gewinnt

BO nutzt ein probabilistisches Modell (GP) um die Unsicherheit zu quantifizieren und gezielt dort zu samplen, wo der erwartete Informationsgewinn maximal ist.

Wann welche Methode?

Entscheidungsbaum:
• ≤ 20 Dimensionen, ≤ 100 Evaluierungen: Bayesian Optimization (Ax, BoTorch)
• ≤ 100 Dimensionen, ≥ 100 Evaluierungen: CMA-ES oder TPE
• > 100 Dimensionen: Random Search + Early Stopping, oder Population-Methoden
• Parallele Evaluierungen (HPC): Ax mit qNEI, oder Populationsmethoden

No Free Lunch

Es gibt keinen universell besten Algorithmus — aber mit Annahmen über die Problemstruktur (Glattheit, Konvexität, Rauschigkeit) können wir informierte Entscheidungen treffen.

from ax.service.ax_client import AxClient
from ax.service.utils.instantiation import ObjectiveProperties

# 1. Experiment definieren
ax_client = AxClient(verbose_logging=False)
ax_client.create_experiment(
    parameters=[
        {"name": "lr", "type": "range", 
         "bounds": [1e-5, 1e-1], "log_scale": True},
        {"name": "batch_size", "type": "choice",
         "values": [16, 32, 64, 128, 256]},
        {"name": "weight_decay", "type": "range",
         "bounds": [1e-6, 1e-2], "log_scale": True},
        {"name": "dropout", "type": "range",
         "bounds": [0.0, 0.5]},
    ],
    objectives={"val_loss": ObjectiveProperties(minimize=True)},
    # Optional: Outcome Constraints
    outcome_constraints=["training_time <= 3600"],
)

# 2. Optimierungsloop
for i in range(25):
    params, trial_index = ax_client.get_next_trial()
    
    # Dein Training hier...
    result = train_model(**params)
    
    ax_client.complete_trial(
        trial_index=trial_index,
        raw_data={
            "val_loss": result["val_loss"],
            "training_time": result["time"],
        }
    )

# 3. Ergebnisse

Tipps aus der Praxis:
• Nutze log_scale=True für Lernraten und Weight Decay
• Starte mit wenigen Parametern, erweitere schrittweise
• Ax's get_next_trial() nutzt intern qNEI für parallele Settings
• Für sehr viele Parameter (>20): Erst Random Search, dann BO auf Top-5 Parameter
