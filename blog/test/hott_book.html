1. Types & Elements

In HoTT, the basic objects are types and elements. Think of a type as a space, and elements as points in that space. Every element belongs to exactly one type.

Unlike set theory where everything is a set and ∈ is a global relation, in type theory a : A is a judgment — it's part of how a was constructed. You can't ask "is true = 1ℕ?" because they live in different types.

Contexts & Judgments

A judgment takes the form Γ ⊢ a : A, meaning "in context Γ, the term a has type A." There are four kinds:

Γ ⊢ A type — "A is a well-formed type"
Γ ⊢ a : A — "a is an element of A"
Γ ⊢ A ≡ B type — "A and B are judgmentally equal types"
Γ ⊢ a ≡ b : A — "a and b are judgmentally equal elements"

Type Families (Dependent Types)

A type family B over A is a type B(x) that varies as x ranges over A. Think of it as a fiber bundle: over each point x in the base space A, there's a fiber B(x).

2. Dependent Function Types (Π-Types)

A dependent function f : Π(x:A) B(x) assigns to each x : A an element f(x) : B(x). The output type can depend on the input!

Think of a dependent function as a section of a fiber bundle. For each point in the base, you pick exactly one point in the fiber above it. An ordinary function A → B is the special case where all fibers are the same type B.

The Four Rules

Formation:
If B(x) is a type for x:A,
then Π(x:A) B(x) is a type.
Introduction (λ):
If b(x) : B(x) for x:A,
then λx.b(x) : Π(x:A) B(x)
Elimination (eval):
If f : Π(x:A) B(x),
then f(x) : B(x) for x:A
Computation (β+η):
β: (λy.b(y))(x) ≡ b(x)
η: λx.f(x) ≡ f

Composition of Functions

Given f : A → B and g : B → C, we get g∘f : A → C defined by (g∘f)(x) := g(f(x)).

3. The Natural Numbers ℕ

ℕ is the archetypal inductive type. It has two constructors: 0 and succ. Its induction principle lets us define functions and prove properties by specifying behavior at 0 and at successors.

Pattern Matching

Instead of explicitly invoking ind_ℕ, we can define functions by pattern matching:

add(m, 0) := m -- base case
add(m, succ(n)) := succ(add(m, n)) -- inductive step

This gives us the judgmental equalities: m + 0 ≡ m and m + succ(n) ≡ succ(m + n).

But note: 0 + n = n requires a proof by induction — it's an identification, not a judgmental equality!

4. Identity Types — Paths in Space

The identity type a =_A x is the type of identifications (paths) from a to x in A. It's inductively generated by refl_a : a = a.

Think of a type as a topological space. Elements are points. An identification p : x = y is a path from x to y. There might be many different paths! The identity type x = y is itself a space — the space of all paths from x to y.

The Groupoid Structure

Every type has the structure of an ∞-groupoid:

  • Reflexivity: refl_x : x = x
  • Symmetry (inverse): If p : x = y, then p⁻¹ : y = x
  • Transitivity (concat): If p : x = y and q : y = z, then p · q : x = z
  • Associativity: (p · q) · r = p · (q · r)
  • Unit laws: refl · p = p, p · refl = p
  • Inverse laws: p⁻¹ · p = refl, p · p⁻¹ = refl

All these "equalities" are themselves identifications — paths between paths!

Transport

Given a type family B over A and a path p : x = y, we get tr_B(p) : B(x) → B(y).

Transport moves elements along paths in the base. It's like parallel transport in differential geometry!

5. Equivalences

An equivalence f : A ≃ B is a function with both a section (right inverse) and a retraction (left inverse). Equivalent types are "the same" for all mathematical purposes.

🎮 Play: Equivalence Explorer

See how equivalences biject points while preserving all structure.

Bi-invertible Maps

is-equiv(f) := sec(f) × retr(f)
where sec(f) = Σ(g:B→A) f∘g ~ id
retr(f) = Σ(h:B→A) h∘f ~ id

Why not just "has an inverse"? Because has-inverse(f) can be homotopically complex, while is-equiv(f) is always a proposition (has at most one proof).

Fibers of a Map

The fiber of f : A → B at b is: fib_f(b) := Σ(a:A) f(a) = b

Key theorem: f is an equivalence ⟺ all fibers of f are contractible.

6. Contractible Types

A type A is contractible if it has a center c : A such that c = x for all x : A. Contractible types are "singletons up to homotopy."

🎮 Play: Contracting a Space

Watch a contractible space shrink to its center of contraction.

The Fundamental Fact

For any a : A, the type Σ(x:A) a = x is contractible with center (a, refl_a).

This is the basis for the fundamental theorem of identity types: to characterize the identity type of A, find a family B(x) with contractible total space Σ(x:A) B(x).

7. Truncation Levels

Types are stratified by how "complex" their identity types are:

🎮 Play: The Truncation Hierarchy

Level -2: Contractible types — at most one point, trivially
Level -1: Propositions — at most one point (any two elements are equal)
Level 0: Sets — identity types are propositions (equality is a property)
Level 1: 1-Groupoids — identity types are sets
Level k+1: (k+1)-types — identity types are k-types

Examples

  • ∅, 𝟏 are propositions (level -1)
  • ℕ, ℤ, Bool, Fin_k are sets (level 0)
  • The universe U is NOT a set — types can be equivalent in many ways!
  • The circle S¹ is a 1-groupoid (its loop space is ℤ)

8. The Univalence Axiom

The univalence axiom is the crown jewel of HoTT. It says: (A = B) ≃ (A ≃ B). Identity of types is equivalent to equivalence of types. Equivalent types are equal.

In classical math, we constantly "identify" isomorphic structures informally. Univalence makes this formal: if two types are equivalent (same structure, same behavior), then they are literally equal in the universe. Every property of one automatically transfers to the other.

🎮 Play: Univalence in Action

See how equivalent types become identified. The path in the universe connects them.

What Univalence Says

idtoeqv : (A =𝒰 B) → (A ≃ B) Univalence Axiom: idtoeqv is an equivalence. In particular: ua : (A ≃ B) → (A =𝒰 B) — introduction transport(ua(f), x) = f(x) — computation

This means: to give a path between types in the universe is the same as giving an equivalence between them.

Consequences

  • Structure Identity Principle: Equal structures are isomorphic, and isomorphic structures are equal
  • Function extensionality follows from univalence
  • The universe 𝒰 is NOT a set — it has nontrivial paths (autoequivalences)
  • Proofs and constructions automatically respect equivalences

9. Higher Inductive Types — The Circle S¹

Higher inductive types (HITs) let us define types with path constructors, not just point constructors. The simplest interesting example is the circle S¹.

Definition of S¹

S¹ is generated by: • base : S¹ — a point • loop : base = base — a path from base to itself

The circle is freely generated by one point and one loop. This means loop ≠ refl — it's a genuinely new path!

The Fundamental Theorem: π₁(S¹) ≅ ℤ

The loop space of the circle is the integers! Each integer n corresponds to going around the loop n times (negative = backwards).

Ω(S¹, base) ≃ ℤ loop⁰ = refl (stay put) loop¹ = loop (go around once) loop² = loop · loop (go around twice) loop⁻¹ = loop⁻¹ (go backwards once) ...

This is proven using the encode-decode method: define a "winding number" type family over S¹ using transport, then show it characterizes the path space.

The universal cover of S¹ is a helix — each sheet corresponds to an integer. Transport along loop shifts up by 1.
