Fourier Multiplication Algorithm

Modular Addition via Rotation on a Circle (mod P = 113)
Computing: 23 + 47 mod 113 = 70
1
Embedding: Map inputs to sine/cosine
The embedding matrix maps each input a and b onto points on a circle using key frequencies wₖ = 2kπ/P. This produces sin(wₖa), cos(wₖa), sin(wₖb), cos(wₖb) for each key frequency k.
2
Trigonometric Identity: Compose rotations
The attention + MLP layers compute cos(wₖ(a+b)) and sin(wₖ(a+b)) using:

cos(wₖ(a+b)) = cos(wₖa)cos(wₖb) − sin(wₖa)sin(wₖb)
sin(wₖ(a+b)) = sin(wₖa)cos(wₖb) + cos(wₖa)sin(wₖb)
3
Unembedding: Compute logits via rotation by −c
For each candidate output c, compute cos(wₖ(a+b−c)). This is maximized when a+b ≡ c (mod P), since the cosine equals 1 when its argument is 0.
4
Constructive Interference: Sum over frequencies
The model sums cos(wₖ(a+b−c)) across all key frequencies. At the correct answer c* = a+b mod P, all cosines equal 1 and constructively interfere. Everywhere else, they destructively interfere, giving small logits.
5
Result: Softmax produces the answer
After softmax, the logit spike at c* = a+b mod P becomes a near-certain probability. The network has computed modular addition using only Fourier transforms and trigonometric identities.
