🧭 What Is This About?

The big picture: A transformer layer is a function that takes a vector of numbers (a "hidden state") and outputs a new vector. The Jacobian matrix captures how every output dimension changes when you wiggle every input dimension — it's the "derivative" of the whole layer. 🎯 Analogy: If the transformer layer is a machine that reshapes clay, the Jacobian tells you exactly how each point of the clay gets stretched, compressed, and rotated at each moment.

This Coq proof establishes 10 fundamental geometric properties of this Jacobian matrix. These aren't approximations — they're mathematically exact, verified by a computer proof assistant (Coq), meaning no logical step can be wrong. 💡 Why care? These properties tell us that transformer layers have a clean geometric structure: stretching and rotation are independent, volume change comes only from stretching, and the decomposition is unique and orthogonal. This matters for understanding training dynamics, gradient flow, and representational geometry.

§0 The Jacobian Matrix — What It Is

In plain English: Suppose you have a function f that takes a 2D point (x, y) and outputs a new 2D point (u, v). The Jacobian matrix J at a point tells you: "If I nudge x a tiny bit, how much do u and v change? If I nudge y, how much do u and v change?"

Formally: J[i][j] = ∂fᵢ/∂xⱼ — the partial derivative of the i-th output with respect to the j-th input.

Visually: The Jacobian transforms a tiny circle of input perturbations into an ellipse of output perturbations. The symmetric part controls the stretching (ellipse shape), and the antisymmetric part controls the rotation (ellipse orientation). 🎯 Analogy: Think of the Jacobian as a local "lens" — at each point in space, it tells you how the function magnifies, squishes, and twists things nearby.
What you see below: A unit circle (blue) is transformed by the Jacobian into an ellipse. The symmetric part (green) shows pure stretching — no rotation. The antisymmetric part (red) shows pure rotation — no stretching. Together they reconstruct the full transformation.

§1 Jacobian Decomposition: J = J_sym + J_antisym Qed ✓

What this theorem says in plain English: Any matrix J can be split into exactly two parts:
  • J_sym = (J + Jᵀ) / 2 — the symmetric part. This is the "stretching" component. It tells you how the function expands or compresses space along different directions.
  • J_antisym = (J − Jᵀ) / 2 — the antisymmetric part. This is the "rotation" component. It tells you how the function twists space around.
And when you add them back together, you get exactly J. No information is lost, nothing is added. 🎯 Analogy: It's like decomposing any motion of your hand into "opening/closing your fingers" (stretching) and "rotating your wrist" (rotation). Every hand motion is exactly the sum of these two independent motions. 💡 Why this matters for transformers: This means we can separately analyze how a transformer layer stretches representations (which affects information content) and how it rotates them (which affects which directions encode what). These are fundamentally different geometric operations.
Theorem jacobian_decomposition_general : forall (n : nat) (J : Mat), mat_eq n J (mat_add (mat_sym J) (mat_antisym J)). Proof. intros n J i j Hi Hj. unfold mat_add, mat_sym, mat_antisym. field. Qed.
🔍 How the proof works (step by step):

1. intros n J i j Hi Hj — "Let n be any dimension, J any matrix, and i,j any valid indices."

2. unfold mat_add, mat_sym, mat_antisym — "Expand the definitions." Now the goal becomes: J(i,j) = (J(i,j)+J(j,i))/2 + (J(i,j)−J(j,i))/2

3. field — "This is just algebra." The two halves add up: (a+b)/2 + (a−b)/2 = a. Done!

§2 Symmetry & Antisymmetry Properties Qed ✓

What these theorems say:
  • J_symᵀ = J_sym — "The symmetric part is actually symmetric." If you flip it across the diagonal (transpose), nothing changes. Entry (i,j) always equals entry (j,i).
  • J_antisymᵀ = −J_antisym — "The antisymmetric part is actually antisymmetric." If you flip it across the diagonal, every entry gets negated. This forces the diagonal to be zero (since a = −a means a = 0).
🎯 Analogy: A symmetric matrix is like a rubber sheet being stretched — it pulls equally in both directions between any two points. An antisymmetric matrix is like a whirlpool — the force from point A to B is exactly opposite to the force from B to A. 💡 Why this matters: These aren't just naming conventions — they're structural guarantees. The symmetric part can be diagonalized with real eigenvalues (principal stretch directions). The antisymmetric part generates rotations. This is the foundation of continuum mechanics applied to neural networks.
Theorem sym_is_symmetric_general : forall (n : nat) (J : Mat), mat_eq n (mat_transpose (mat_sym J)) (mat_sym J). Proof. intros n J i j Hi Hj. unfold mat_transpose, mat_sym. field. Qed. (* Expanding: transpose of (J(i,j)+J(j,i))/2 = (J(j,i)+J(i,j))/2 = same thing. *) Theorem antisym_is_antisymmetric_general : forall (n : nat) (J : Mat), mat_eq n (mat_transpose (mat_antisym J)) (mat_neg (mat_antisym J)). Proof. intros n J i j Hi Hj. unfold mat_transpose, mat_antisym, mat_neg. field. Qed. (* Expanding: transpose of (J(i,j)−J(j,i))/2 = (J(j,i)−J(i,j))/2 = −(J(i,j)−J(j,i))/2. *)
Reading the bar charts: Each group of bars shows the same matrix entry computed two different ways. For the left chart, the blue bar (J_sym) and green bar (J_symᵀ) should be identical — and they are. For the right chart, the orange bar (J_antisym) and purple bar (−J_antisymᵀ) should be identical — and they are. This is the visual proof.
Geometric view above: The left shows J_sym's effect on a grid — pure stretching, symmetric about principal axes. The right shows J_antisym's effect — pure rotation, no stretching. Notice how the symmetric part creates elliptical distortion while the antisymmetric part creates circular swirling.

§3 Trace = Divergence = Volume Change Qed ✓

What the trace is: The trace of a matrix is just the sum of its diagonal entries: tr(J) = J[0][0] + J[1][1] + J[2][2] + ...

What it means geometrically: The trace equals the divergence of the vector field — it measures how much the transformation expands or compresses volume at a point. Positive trace = expansion (things spread out). Negative trace = compression (things squeeze together). Zero trace = volume-preserving.

The three theorems here say:
  • tr(J) = tr(J_sym) — Volume change comes entirely from the stretching part. Rotation doesn't change volume.
  • tr(J_antisym) = 0 — The rotation part never changes volume. (Because diagonal entries of an antisymmetric matrix are always zero!)
  • tr(cJ) = c · tr(J) — Scaling the transformation scales the volume change proportionally.
🎯 Analogy: Spinning a ball doesn't change its volume — only squeezing or inflating it does. The trace captures only the squeezing/inflating, never the spinning. 💡 For transformers: This means the "information capacity" change of a layer (measured by how much it expands/compresses the representation space) is entirely determined by the symmetric part of its Jacobian. The rotational part just reorients information without changing capacity.
Theorem divergence_from_symmetric_part_general : forall (n : nat) (J : Mat), trace_n n J = trace_n n (mat_sym J). Proof. intros n J. unfold trace_n, mat_sym. apply sum_f_ext. intros i _. field. Qed. (* Key insight: on the diagonal, sym(J)(i,i) = (J(i,i)+J(i,i))/2 = J(i,i). *) (* So summing the diagonal of sym(J) gives the same as summing the diagonal of J. *) Theorem antisym_trace_zero_general : forall (n : nat) (J : Mat), trace_n n (mat_antisym J) = 0. Proof. intros n J. unfold trace_n, mat_antisym. rewrite (sum_f_ext _ (fun _ => 0) n). - apply sum_f_zero. - intros i _. field. (* antisym(J)(i,i) = (J(i,i)−J(i,i))/2 = 0 *) Qed.
Reading the chart: The blue line (tr(cJ)) and yellow dotted line (c·tr(J)) overlap perfectly — trace is linear. The green dashed line (tr(sym(cJ))) also overlaps — divergence comes from the symmetric part. The red line (tr(antisym(cJ))) is always flat at zero — rotation never contributes to volume change. §4 Frobenius Orthogonality Qed ✓

What these theorems say:
  • ⟨J_sym, J_antisym⟩_F = 0 — The symmetric and antisymmetric parts are perpendicular in matrix space. Their Frobenius inner product (which is like a dot product but for matrices) is exactly zero.
What is the Frobenius inner product? Think of "unrolling" a matrix into a long vector (just list all entries in order). The Frobenius inner product is just the ordinary dot product of these unrolled vectors. Formally: ⟨A, B⟩_F = Σᵢ Σⱼ A[i][j] · B[i][j] = tr(AᵀB).

Why is this zero? The proof uses a beautiful swap-and-cancel argument. Each term in the double sum at position (i,k) is (J[k][i]² − J[i][k]²)/4. When you swap i↔k, this becomes (J[i][k]² − J[k][i]²)/4 — the exact negative! So every term cancels with its partner, and the whole sum is zero. 🎯 Analogy: Imagine two arrows in space that are perfectly perpendicular (like North and East). No matter how long each arrow is, their dot product is zero — they share no common direction. Similarly, the stretching part and the rotation part of J live in completely independent "directions" in matrix space. 💡 Why this matters: Because the parts are orthogonal, we get a Pythagorean theorem for matrices: ‖J‖² = ‖J_sym‖² + ‖J_antisym‖². The total "energy" of the Jacobian splits cleanly into stretching energy and rotation energy with no cross-terms. This means you can measure them independently.
Theorem frobenius_sym_antisym_orthogonal : forall (n : nat) (J : Mat), frobenius_inner n (mat_sym J) (mat_antisym J) = 0. Proof. intros n J. unfold frobenius_inner, trace_n, mat_mult, mat_transpose, mat_sym, mat_antisym. (* Rewrite each summand to (J(k,i)² − J(i,k)²)/4 *) rewrite (sum_f_ext _ (fun i => sum_f (fun k => (J k i * J k i - J i k * J i k) / 4) n) n). (* Key: sum S + sum S = 0, because swapping i↔k negates each term *) assert (Hkey : S + S = 0). (* where S is the double sum *) { rewrite (sum_f_swap ...) at 1. rewrite <- sum_f_add. (* Each inner sum: f(i,k) + f(k,i) = 0 by lra *) ... } lra. (* S + S = 0 implies S = 0 *) Qed.
🔍 The swap-and-cancel argument visualized:

Consider the matrix of terms f(i,k) = (J[k][i]² − J[i][k]²)/4. This matrix is antisymmetric: f(i,k) = −f(k,i). When you sum all entries of an antisymmetric matrix, every above-diagonal term cancels with its below-diagonal partner, and diagonal terms are zero. So the total sum must be zero.

Reading the left chart: The bars show ‖J‖²_F, ‖J_sym‖²_F, ‖J_antisym‖²_F, their sum, and the inner product. The inner product bar (rightmost) should be zero (green). The Pythagorean identity says the first bar equals the fourth bar.

Reading the right chart: This shows the individual terms f(i,k) = (J[k][i]² − J[i][k]²)/4 in the Frobenius inner product sum. Notice how terms above the diagonal (blue) are exactly negated by terms below the diagonal (red), and diagonal terms are zero. This is the cancellation that makes the proof work.
Geometric view above: In the "matrix space" (here projected to 2D), J_sym and J_antisym are shown as perpendicular vectors. J is their sum (the diagonal of the rectangle). The right angle confirms orthogonality. Drag the §1 sliders to see how the vectors change while always staying perpendicular.

§5 Idempotence & Cross-Annihilation Qed ✓

What these four theorems say in plain English:
  • sym(sym(J)) = sym(J) — "Symmetrizing something that's already symmetric does nothing." If you've already extracted the stretching part, extracting it again gives the same thing. This is called idempotence.
  • antisym(antisym(J)) = antisym(J) — Same for the rotation part.
  • sym(antisym(J)) = 0 — "The stretching part of a pure rotation is zero." If you take something that's purely rotational and ask 'what's the stretching component?', the answer is nothing.
  • antisym(sym(J)) = 0 — "The rotation part of a pure stretch is zero." Vice versa.
Together, these four properties say that sym and antisym are complementary orthogonal projections. They're like two filters: one passes only horizontal light (stretching), the other only vertical light (rotation). Applying the same filter twice doesn't change anything, and applying the wrong filter blocks everything. 🎯 Analogy: Think of polarized sunglasses. A horizontal polarizer (sym) passes horizontal light and blocks vertical. Putting two horizontal polarizers in a row is the same as one (idempotent). Putting a horizontal polarizer after a vertical one (cross-annihilation) blocks all light. 💡 Why this matters: This guarantees the decomposition is unique and clean. There's exactly one way to split J into stretching and rotation, and the two parts don't "leak" into each other. For transformer analysis, this means measurements of strain and rotation are unambiguous.
Theorem sym_idempotent_general : forall (n : nat) (J : Mat), mat_eq n (mat_sym (mat_sym J)) (mat_sym J). Proof. intros n J i j Hi Hj. unfold mat_sym. field. Qed. (* sym(sym(J))(i,j) = ((J(i,j)+J(j,i))/2 + (J(j,i)+J(i,j))/2) / 2 = (J(i,j)+J(j,i)) / 2 = sym(J)(i,j) ✓ *) Theorem sym_of_antisym_zero_general : forall (n : nat) (J : Mat), mat_eq n (mat_sym (mat_antisym J)) mat_zero. Proof. intros n J i j Hi Hj. unfold mat_sym, mat_antisym, mat_zero. field. Qed. (* sym(antisym(J))(i,j) = ((J(i,j)−J(j,i))/2 + (J(j,i)−J(i,j))/2) / 2 = 0 / 2 = 0 ✓ *)
🔍 Why does field solve these instantly?

After unfolding definitions, each goal is a rational arithmetic identity. For example, sym(antisym(J))(i,j) expands to ((a−b)/2 + (b−a)/2)/2 where a=J(i,j) and b=J(j,i). This simplifies to 0/2 = 0. The field tactic handles this automatically.

Reading the bar chart: Each bar shows the maximum absolute error for one of the four properties. All bars should be at machine-zero (≈10⁻¹⁶). The y-axis is logarithmic — if a bar is visible above 10⁻¹⁰, something is wrong.

Reading the matrices: sym(antisym(J)) and antisym(sym(J)) are displayed numerically. Every entry should be 0.000 (or very close to it).

§6 Shear Tensor is Trace-Free Qed ✓

What is the shear tensor? We already know J_sym captures all the stretching. But stretching has two sub-components:
  • Uniform expansion/compression — scaling everything equally in all directions. This is captured by (tr(J_sym)/n) · I, a scalar multiple of the identity.
  • Pure shear — stretching more in some directions than others, but with no net volume change. This is the shear tensor S = J_sym − (tr(J_sym)/n) · I.
The theorem says: tr(S) = 0. The shear tensor has zero trace, meaning it causes zero net volume change. All volume change is captured by the (tr/n)·I part.

The proof is elegant: tr(S) = tr(J_sym) − tr((tr(J_sym)/n)·I) = tr(J_sym) − (tr(J_sym)/n)·n = tr(J_sym) − tr(J_sym) = 0. 🎯 Analogy: Imagine squeezing a balloon. The shear is like squeezing it from the sides (it gets taller but thinner — shape changes but volume doesn't). The expansion part is like inflating it (volume changes but shape stays the same). The shear tensor isolates the shape-change-only part. 💡 For transformers: This three-way decomposition (rotation + uniform scaling + shear) gives a complete geometric picture: J = rotation + (volume change rate)·I/n + pure shape change. Each component is independently measurable and has a distinct geometric meaning.
Definition mat_shear (n : nat) (A : Mat) : Mat := fun i j => mat_sym A i j - (trace_n n (mat_sym A) / INR n) * (if Nat.eq_dec i j then 1 else 0). Theorem shear_trace_free_general : forall (n : nat) (J : Mat), (n > 0)%nat -> trace_n n (mat_shear n J) = 0. Proof. intros n J Hn. unfold trace_n, mat_shear. (* Diagonal entries: sym(J)(i,i) − tr(sym(J))/n *) (* Sum = tr(sym(J)) − n · tr(sym(J))/n = 0 *) ... field. exact HnR. (* need n ≠ 0 for division *) Qed.
Reading the three heatmaps:
  • Left (J_sym): The full symmetric part, with nonzero trace (shown in title).
  • Middle (Shear S): The trace-free part. Notice the diagonal values have been adjusted so they sum to zero.
  • Right (Isotropic): The (tr/n)·I part — a diagonal matrix with equal entries. This is the pure volume-change component.
Together: J_sym = Shear + Isotropic. The shear changes shape, the isotropic part changes volume.
Above: A circle is transformed by J_sym (left), by the shear alone (middle), and by the isotropic part alone (right). The shear distorts the circle into an ellipse without changing area. The isotropic part scales the circle uniformly. Together they give the full J_sym effect.

§7 Residual Connection: J = I + Δ Qed ✓

What is a residual connection? In a transformer, each layer doesn't completely replace the hidden state — it adds to it: h_new = h_old + Δ(h_old). This "skip connection" is crucial for training deep networks.

At the Jacobian level: The Jacobian of h_new with respect to h_old is J = I + J_Δ, where I is the identity matrix (from the "pass-through" of h_old) and J_Δ is the Jacobian of the residual block Δ.

Two theorems:
  • tr(I + J_Δ) = n + tr(J_Δ) — The divergence of the full layer is the dimension n (from the identity) plus the divergence of the residual block. The identity contributes a "baseline expansion" of n.
  • antisym(I + J_Δ) = antisym(J_Δ) — The rotation comes entirely from the residual block. The identity is perfectly symmetric, so it contributes zero rotation.
🎯 Analogy: Imagine you're walking forward (the identity — you keep going where you were going) while a wind blows you sideways (the residual Δ). Your total velocity is your walking speed plus the wind. The "rotation" (change of direction) comes entirely from the wind, not from your steady walking. 💡 For transformers: This explains why residual connections stabilize training. The identity provides a guaranteed baseline divergence of n (no information loss), and the residual block only needs to learn small corrections. The rotation (which reorients representations) is entirely learned.
Theorem residual_divergence_general : forall (n : nat) (J_delta : Mat), trace_n n (mat_add mat_id J_delta) = INR n + trace_n n J_delta. Proof. intros n J_delta. rewrite trace_additive_general. (* tr(A+B) = tr(A) + tr(B) *) rewrite trace_id_general. (* tr(I) = n *) reflexivity. Qed. Theorem residual_antisym_general : forall (n : nat) (J_delta : Mat), mat_eq n (mat_antisym (mat_add mat_id J_delta)) (mat_antisym J_delta). Proof. intros n J_delta i j Hi Hj. unfold mat_antisym, mat_add, mat_id. (* Case split: if i=j, both sides are 0. If i≠j, the 1s cancel. *) destruct (Nat.eq_dec i j); destruct (Nat.eq_dec j i); field. Qed.
🔍 Why does the identity contribute no rotation?

The identity matrix I has I[i][j] = 1 if i=j, 0 otherwise. It's perfectly symmetric: I = Iᵀ. So antisym(I) = (I − Iᵀ)/2 = 0. When we compute antisym(I + J_Δ) = antisym(I) + antisym(J_Δ) = 0 + antisym(J_Δ), the identity drops out completely.

Left chart: As you increase the Δ scale, tr(I+Δ) (blue) and n+tr(Δ) (green dashed) always overlap perfectly — confirming the divergence formula.

Right chart: A stacked bar showing how the total divergence breaks down: the blue portion is always exactly n (from the identity), and the green portion is tr(Δ) (from the residual block). The identity provides a stable "floor" of divergence.

§8 Trace Cyclicity: tr(ABC) = tr(BCA) = tr(CAB) Qed ✓

What this theorem says: You can cyclically permute matrices inside a trace without changing the result. tr(ABC) = tr(BCA) = tr(CAB). Note: you can NOT arbitrarily reorder them — tr(ABC) ≠ tr(BAC) in general. Only cyclic permutations work.

How the proof works:
  1. Start with the 2-matrix version: tr(XY) = tr(YX). This follows from swapping the summation order in the double sum Σᵢ Σₖ X[i][k]·Y[k][i].
  2. For three matrices: tr(A·(BC)) = tr((BC)·A) by the 2-matrix version (treating BC as one matrix).
  3. Then use associativity: (BC)A = B(CA), so tr(B(CA)) = tr((CA)B) = tr(C(AB)) by applying the 2-matrix version again.
🎯 Analogy: Imagine three people A, B, C sitting in a circle, each passing a number to the next person who multiplies it. The total product around the circle is the same no matter who you start counting from — because it's a circle! 💡 For transformers: When analyzing compositions of multiple layers (J₃ · J₂ · J₁), trace cyclicity lets us compute the divergence starting from any layer. This is essential for analyzing multi-layer gradient flow and information propagation.
Theorem trace_cyclic_3_general : forall (n : nat) (A B C : Mat), trace_n n (mat_mult n A (mat_mult n B C)) = trace_n n (mat_mult n C (mat_mult n A B)). Proof. intros n A B C. (* Step 1: tr(A(BC)) = tr((BC)A) by tr(XY) = tr(YX) *) rewrite (trace_mult_commutative_general n A (mat_mult n B C)). (* Step 2: tr((BC)A) = tr(B(CA)) by associativity *) ... (* Step 3: tr(B(CA)) = tr((CA)B) by tr(XY) = tr(YX) *) rewrite (trace_mult_commutative_general n B (mat_mult n C A)). (* Step 4: tr((CA)B) = tr(C(AB)) by associativity *) ... reflexivity. Qed.

§9 Orthogonal Similarity Invariance Qed ✓

What this theorem says: If Q is an orthogonal matrix (QQᵀ = I, meaning Q is a rotation/reflection), then tr(QᵀJQ) = tr(J). The trace (divergence) doesn't change when you rotate your coordinate system.

How the proof works:
  1. tr(Qᵀ(JQ)) = tr((JQ)Qᵀ) by trace cyclicity (tr(XY) = tr(YX))
  2. tr((JQ)Qᵀ) = tr(J(QQᵀ)) by associativity
  3. tr(J(QQᵀ)) = tr(J·I) = tr(J) because QQᵀ = I
🎯 Analogy: Imagine measuring the temperature in a room. You can set up your thermometer facing North, or rotate it to face East — the temperature reading doesn't change. Similarly, the trace (divergence) is a "scalar" quantity that doesn't depend on which coordinate axes you use. Rotating your axes (via Q) doesn't affect it. 💡 For transformers: Different attention heads may operate in different rotated subspaces. This theorem guarantees that the divergence (information capacity change) is the same regardless of which basis you measure it in. It's an intrinsic property of the layer, not an artifact of representation.
Theorem trace_orthogonal_similarity : forall (n : nat) (Q J : Mat), mat_eq n (mat_mult n Q (mat_transpose Q)) mat_id -> trace_n n (mat_mult n (mat_transpose Q) (mat_mult n J Q)) = trace_n n J. Proof. intros n Q J HQ. (* Step 1: tr(Qᵀ(JQ)) = tr((JQ)Qᵀ) by tr(XY) = tr(YX) *) rewrite trace_mult_commutative_general. (* Step 2: tr((JQ)Qᵀ) = tr(J(QQᵀ)) by associativity *) assert (Hassoc : ...). { apply mat_eq_trace_ext. apply mat_mult_assoc_general. } rewrite Hassoc. (* Step 3: tr(J(QQᵀ)) = tr(J·I) since QQᵀ = I *) assert (HJI : ...). { apply mat_eq_trace_ext. intros i j Hi Hj. unfold mat_mult. f_equal. apply HQ; assumption. } rewrite HJI. (* Step 4: tr(J·I) = tr(J) by right-identity *) unfold trace_n, mat_mult, mat_id. apply sum_f_ext. intros i Hi. apply sum_f_delta_right. exact Hi. Qed.
🔍 The four-step chain in detail:

Step 1 (Cyclicity): tr(QᵀJQ) = tr(Qᵀ · (JQ)) = tr((JQ) · Qᵀ). We used trace_mult_commutative_general — the same theorem from §8 that says tr(XY) = tr(YX).

Step 2 (Associativity): tr((JQ)Qᵀ) = tr(J(QQᵀ)). We used mat_mult_assoc_general to re-bracket the product.

Step 3 (Orthogonality hypothesis): Since QQᵀ = I (our assumption), we replace QQᵀ with I to get tr(J·I).

Step 4 (Right identity): J·I = J, so tr(J·I) = tr(J). Done!

Right canvas: The 2×2 matrix J transforms a unit circle into an ellipse (blue). As you rotate Q (change θ), the rotated-basis version QᵀJQ transforms the same circle into a different-looking ellipse (green) — but the trace (sum of eigenvalues, related to the ellipse's "total stretch") stays the same.

§10 Grand Summary: All 10 Properties Qed ✓

The final theorem transformer_jacobian_geometry_complete bundles all 10 properties into a single conjunction. In Coq, a conjunction A ∧ B ∧ C ∧ ... means ALL of these must be true simultaneously. The proof simply references each individual theorem we proved above.

What this means: For ANY matrix J of ANY dimension n, ALL of the following hold simultaneously:
  1. J decomposes into symmetric + antisymmetric parts
  2. The symmetric part is genuinely symmetric
  3. The antisymmetric part is genuinely antisymmetric
  4. The two parts are Frobenius-orthogonal
  5. Divergence comes only from the symmetric part
  6. The antisymmetric part has zero trace
  7. The symmetric projector is idempotent
  8. The antisymmetric projector is idempotent
  9. sym(antisym(J)) = 0 (cross-annihilation)
  10. antisym(sym(J)) = 0 (cross-annihilation)
