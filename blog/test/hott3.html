Key insight: In classical type theory, equality is boring (either things are equal or not). In HoTT, equality is RICH — there can be many DIFFERENT ways two things are equal, just as there can be many different paths between two points in a space!

2. Types as Spaces, Terms as Points

In HoTT, every type is thought of as a topological space. The terms (inhabitants) of that type are points in the space. We can manipulate spaces directly without first having to develop point-set topology.

Different types visualized as spaces with their inhabitants (points)

💡 Intuition

Think of a type like Bool as a space with exactly two points: true and false. The type ℕ (natural numbers) is a discrete space with countably many points: 0, 1, 2, 3, ... But the type S¹ (the circle) is a connected space where every point is connected to every other by a path!

The Hierarchy of h-levels

Types are classified by how "interesting" their path structure is:

  • (-2) Contractible types: A single point. Only one element, one path, one everything. (e.g., the unit type ⊤)
  • (-1) Propositions (mere props): Either empty or contractible. At most one element, and any two elements are equal. (e.g., isEmpty(A))
  • (0) Sets: Any two equality proofs between the same elements are equal. No interesting higher path structure. (e.g., ℕ, Bool)
  • (1) Groupoids: Paths can differ, but paths-between-paths are unique. (e.g., the universe of sets)
  • (n) n-Groupoids: Interesting structure up to level n, then trivial.
  • (∞) General types: Interesting path structure at ALL levels! (e.g., the circle S¹)

3. Identity Types as Paths

The revolutionary idea: given a b : A, the identity type a =A b is itself a type — the space of paths from a to b. A term p : a =A b is a specific path.

Paths between points represent proofs of equality. Multiple distinct paths = multiple proofs!

Path deformation: 50%
refl(a) : a =A a — The constant path (reflexivity). Every point has a trivial loop. p : a =A b — A path from a to b (proof that a equals b) p⁻¹ : b =A a — The reverse path (symmetry of equality) p · q : a =A c — Composition of p: a=b and q: b=c (transitivity)
🔑 Why This Matters

In classical foundations (ZFC set theory), if a = b, that's the end of the story. There's only ONE way things can be equal. But in HoTT, the identity type a = b can have many inhabitants — many different paths/proofs of equality. This is what makes HoTT capture the full richness of topology!

4. Path Composition & Groupoid Structure

Paths form an ∞-groupoid structure. You can compose paths, reverse them, and these operations satisfy the groupoid laws up to homotopy (not strictly!).

Path composition: p · q gives a path from a to c via b

Animation progress: 0%

The Groupoid Laws (up to homotopy)

For paths p : a = b, q : b = c, r : c = d:

  • Left unit: refl(a) · p = p (but only up to a 2-path!)
  • Right unit: p · refl(b) = p
  • Inverse: p · p⁻¹ = refl(a) and p⁻¹ · p = refl(b)
  • Associativity: (p · q) · r = p · (q · r) (up to a 2-path!)

The phrase "up to homotopy" is crucial: these equalities are themselves paths (2-paths), not strict equalities. This gives rise to the full ∞-groupoid structure.

5. Higher Paths: Homotopies Between Homotopies

Since p =(a=b) q is itself a type, we get paths between paths — these are 2-dimensional surfaces (homotopies). And paths between those form 3-cells, and so on to infinity!

🌀 The Infinite Tower

Imagine two rubber bands stretched between the same two nails (two paths p, q from a to b). A 2-path is a way to continuously deform one rubber band into the other. But there might be DIFFERENT ways to do this deformation — and a 3-path records how to deform one deformation into another. This structure goes on forever, capturing the full homotopy type of a space.

6. The Univalence Axiom

Voevodsky's Univalence Axiom states: (A ≃ B) ≃ (A =𝒰 B)

In words: two types are equal (in the universe) if and only if they are equivalent. Equivalence IS equality for types!

The universe 𝒰 as a space whose points are types. A path in 𝒰 IS an equivalence between types!

What Univalence Gives Us

Without univalence: We might have two types that are "the same" (isomorphic) but technically different. We'd need to manually transport constructions between them.

With univalence: If A ≃ B, then A = B in the universe. Any property of A automatically holds for B. This is the principle of structural invariance — mathematics should be invariant under equivalence.

The universe 𝒰 has terms that are themselves types: A : 𝒰 Univalence: (A =𝒰 B) ≃ (A ≃ B) Example: Bool ≃ Bool via the swap map (true↔false). By univalence, this gives a non-trivial loop: Bool =𝒰 Bool So the universe is NOT a set — it has interesting path structure!
🎯 Practical Consequence

Suppose you prove a theorem about lists. Then you discover that for your application, balanced binary trees would be more efficient, and lists ≃ trees (as types of sequences). Univalence says your theorem automatically transfers to trees — no extra work needed! The equivalence IS an equality.

7. Higher Inductive Types: The Circle S¹

Higher Inductive Types (HITs) let us define spaces by specifying not just points but also paths, 2-paths, etc. The simplest example is the circle S¹:

S¹ is defined by: • base : S¹ (one point constructor) • loop : base =S¹ base (one path constructor — a loop!) That's it! From these two generators, the full circle emerges. The loop can be composed with itself: loop · loop · loop ... giving winding numbers!

The circle S¹ as a Higher Inductive Type. The winding number counts how many times loop is composed.

Why HITs are Revolutionary

In classical type theory, you can only define types by specifying their points. HITs let you also specify paths and higher paths as constructors. This means you can directly define topological spaces like:

  • S¹ (Circle): one point + one loop
  • S² (Sphere): one point + one 2-cell (surface filling a loop)
  • Torus: one point + two loops + one 2-cell
  • Quotient types: add paths to identify elements
  • Truncations: add higher paths to kill higher homotopy

8. Fundamental Groups via HoTT

The loop space Ω(A, a) = (a =A a) is the type of all loops at point a. For the circle, we can prove in HoTT that Ω(S¹, base) ≃ ℤ — the fundamental group of the circle is the integers!

The loop space of S¹: each integer corresponds to a winding number (how many times you go around)

The Encode-Decode Method

To prove Ω(S¹) ≃ ℤ in HoTT, we use the encode-decode method:

  1. Define a cover: A type family code : S¹ → 𝒰 where code(base) = ℤ
  2. Encode: Map loops to integers by counting winding number
  3. Decode: Map integers back to loops by iterating loop
  4. Show round-trip: encode ∘ decode = id and decode ∘ encode = id

This is a fully constructive proof that works in a proof assistant like Coq or Agda!

Summary: The Big Picture

Homotopy Type Theory gives us a foundation where:

  • Types are spaces — with rich geometric structure
  • Equality is paths — proofs of equality carry information
  • Equivalent types are equal — via the Univalence Axiom
  • Spaces can be defined synthetically — via Higher Inductive Types
  • Everything is constructive — computable and formalizable in proof assistants

This unification of logic, type theory, and topology opens new doors in both mathematics and computer science, providing a framework where abstract homotopy theory becomes directly expressible in a programming language.
