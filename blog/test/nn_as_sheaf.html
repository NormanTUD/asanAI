Neural Networks as Sheaves

How an optimizer glues local linear pieces into a global function approximation

1. The Core Idea

A sheaf is a mathematical structure that builds global objects by consistently gluing together local data defined on overlapping open sets. A neural network with ReLU activations does something strikingly analogous:

  • Each neuron computes an affine function (local linear piece).
  • Each ReLU activation bends that piece — it "folds" the domain into regions where the neuron is active or silent.
  • The optimizer (e.g. SGD / Adam) acts as the gluing morphism, adjusting weights so that all local pieces are compatible on overlaps and together approximate a target function.

Sheaf Intuition: The optimizer doesn't see the global function directly. It nudges each local affine piece (section) and each ReLU fold (restriction map) so that when you glue them all via the output layer, the result is globally consistent with the data. This is precisely the sheaf condition: local data, globally compatible.

6. The Sheaf Perspective — Recap

Sheaf Concept Neural Network Analogue
Open cover {Uᵢ} Regions where each ReLU neuron is active vs. inactive
Local section on Uᵢ Affine function wⱼx + bⱼ (or 0) on each region
Restriction map ReLU activation (restricts to zero on the negative part)
Gluing axiom Output layer's weighted sum assembles a global function
Gluing morphism / cocycle Optimizer adjusting weights to ensure global consistency with data
Key Takeaway: A ReLU network is a machine for constructing piecewise-linear sheaves. Each training step refines the local sections and their gluing data until the global section (the network's output) faithfully represents the target function. More neurons = finer cover = better approximation.
