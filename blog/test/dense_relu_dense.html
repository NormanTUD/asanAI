2D: Vektor [x₁, x₂] → Dense → ReLU → Dense

Was passiert: Ein 2D-Punkt wird durch eine Matrix transformiert (Rotation, Scherung, Skalierung), verschoben (Translation), durch ReLU gefaltet, und dann nochmal transformiert.
Ergebnis: Der 2D-Raum wird in Regionen aufgeteilt und stückweise linear abgebildet!

h = ReLU(z) = [max(0,z₁), max(0,z₂)]
y = W₂ · h + b₂    wobei W₂ ∈ ℝ²ˣ², b₂ ∈ ℝ²

📖 Was Dense → ReLU → Dense wirklich macht

Die Grundidee

Dense (Affine Transformation):

z = W·x + b

→ Rotation, Skalierung, Scherung, Translation


ReLU (Rectified Linear Unit):

h = max(0, z)

→ "Faltet" den Raum: alles Negative wird auf 0 gesetzt


Dense (nochmal Affine):

y = W·h + b

→ Nochmal drehen, strecken, verschieben

Was kann es pro Dimension?

1D Stückweise lineare Funktion mit 1 Knick pro Neuron
2D Raum wird entlang von Linien gefaltet → stückweise lineare Abbildung
3D Raum wird entlang von Ebenen gefaltet
nD Raum wird entlang von Hyperebenen gefaltet. n Neuronen → bis zu 2ⁿ lineare Regionen!

Affine Transformationen erklärt

Skalierung: W = [[s₁, 0], [0, s₂]] → streckt/staucht Achsen

Rotation: W = [[cos θ, -sin θ], [sin θ, cos θ]] → dreht um Ursprung

Scherung: W = [[1, k], [0, 1]] → verschiebt proportional

Translation: +b → verschiebt alles um b

Kombination: W·x + b kann all das gleichzeitig!

Warum ist ReLU so mächtig?

Ohne ReLU: Dense → Dense = eine einzige affine Transformation (W₂·W₁·x + W₂·b₁ + b₂)

Mit ReLU dazwischen: Der Raum wird in Regionen aufgeteilt, und in jeder Region gilt eine ANDERE lineare Funktion!

→ Das ist der Universal Approximation Theorem in Aktion: genug Neuronen → beliebige Funktion approximierbar!

Maximale Anzahl linearer Regionen

Für ein Netzwerk mit Eingabedimension d und n ReLU-Neuronen:

Max. Regionen = Σᵢ₌₀ᵈ C(n, i)

Für n ≥ d: ≈ nᵈ/d! (polynomiell in n, exponentiell in d)

Für d = 1, n Neuronen: n+1 Regionen (n Knickstellen)

Für d = 2, n Neuronen: bis zu 1 + n + n(n-1)/2 Regionen
