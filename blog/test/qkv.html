🔍 Transformer Attention: Q, K, V Interactive Explorer

Build your geometric intuition for how Queries, Keys, and Values work inside transformer attention — from simple 2D to full multi-head attention.

Step 1: Token Embeddings in Space

Every token in a transformer is represented as a vector in a high-dimensional space. Here we use 2D so you can see it directly. Each axis represents a learned semantic dimension. Think of them as "concepts" — e.g., one axis might correlate with "animal-ness", another with "size".

Key insight: Tokens that are semantically similar (cat, dog) live close together in embedding space. The embedding is the starting point — attention will transform these positions via learned linear maps.

Step 2: Q, K, V Linear Projections

Each embedding is multiplied by three learned weight matrices to produce Query (Q), Key (K), and Value (V) vectors. These are linear transformations — geometrically they are rotations, scalings, and shears of the embedding space.
Intuition:
• Q (Query) = "What am I looking for?" — rotates the token into a "question" space
• K (Key) = "What do I contain?" — rotates the token into a "label" space
• V (Value) = "What information do I carry?" — rotates into a "content" space

Attention scores come from Q·K similarity. The output is a weighted sum of V vectors.

Watch how: The Q and K projections rotate/scale tokens differently. Two tokens might be close in embedding space but far apart in Q-K space (or vice versa). The V projection determines what "content" each token carries to the output. Try making W_Q a pure rotation vs. a stretch — see how it changes which tokens align!

Step 3: Dot-Product Attention Scores

For each query token, we compute its dot product with all key tokens. The dot product measures alignment — how much two vectors point in the same direction. Geometrically: = projection of q onto k × magnitudes.

1.41 (√2)
Geometric view: The query vector (red arrow) is projected onto each key vector (teal arrows). The longer the projection and the more aligned the directions, the higher the attention score. The scaling prevents scores from becoming too large in high dimensions.

Step 4: Softmax — From Scores to Weights

Raw dot-product scores are converted to a probability distribution via softmax. This creates attention weights that sum to 1. Higher temperature → more uniform; lower → more peaked.
cat dog car runs
1.00
Key insight: Softmax is a "soft" argmax. At low temperature it approaches one-hot (hard attention). At high temperature it approaches uniform distribution (all tokens attend equally). Real transformers use which is moderate scaling.

Step 5: Value Aggregation — The Output

The final attention output for each query position is a weighted sum of Value vectors, where the weights come from softmax. Geometrically, the output vector is pulled toward the V vectors of highly-attended tokens.
cat dog car runs
Geometric intuition: The output (purple star) is a convex combination of V vectors. It lies inside the convex hull of the V vectors, pulled toward those with highest attention weight. This is how attention "mixes" information from different positions — it doesn't copy one token, it blends content from multiple tokens based on Q-K similarity.

Step 6: Full 3D Multi-Head Attention

In real transformers, embeddings are high-dimensional (768, 1024, etc.) and attention uses multiple "heads" that each learn different Q, K, V projections — attending to different relationship types simultaneously. Multi-head intuition: Each head "looks" at the sequence from a different angle. Head 1 might attend based on syntactic relationships, Head 2 based on semantic similarity. The final output concatenates (or averages) all heads, giving the model a rich, multi-faceted understanding of token relationships. This is why transformers are so powerful — they don't just compute one type of attention, they compute many in parallel.
