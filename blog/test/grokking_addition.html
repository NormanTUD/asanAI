🧠 Grokking: How a Neural Network Learns to Add

What's This Paper About?

The Big Question: Researchers trained a tiny neural network (a transformer) to do modular addition — like clock arithmetic. Something weird happened: the network first memorized all the answers (like a student cramming), then suddenly understood the pattern. This sudden "aha!" moment is called grokking.


The Discovery: They cracked open the network and found it learned a beautiful algorithm using circles, waves, and trigonometry — converting addition into rotation!


What you'll learn here: Step by step, interactively, how the network turns numbers into points on circles, uses trig identities to add them, and reads off the answer using wave interference.

Step 0: What is Modular Addition?

Modular addition is like a clock. On a 12-hour clock, 10 + 5 = 3 (not 15). The paper uses mod 113 (a prime number), so numbers wrap around at 113.

Step 1: Mapping Numbers to a Circle (Embedding)

The network's first trick: it maps each number to a point on a circle using sine and cosine. For a number a and frequency k, it computes:

x = cos(2πk·a / P)     y = sin(2πk·a / P)

This is like placing the number at an angle on a circle. Different frequencies k give different "views" of the same number.

Abitur Connection: Remember the unit circle? cos(θ) gives the x-coordinate and sin(θ) gives the y-coordinate of a point at angle θ. Here, the angle is proportional to the number a.

Step 2: Why Fourier? Sparsity in Frequency Space

The network doesn't use ALL possible frequencies — it picks just 5 "key frequencies" (out of 56 possible). In the paper's main model, these are k ∈ {14, 35, 41, 42, 52}.

Think of it like a radio: out of all possible stations, the network tunes into just 5 channels that together give it enough information.

Step 3: The Core Trick — Trig Identities Compute Addition!

Here's the magical insight: the network uses the cosine addition formula to turn multiplication of embeddings into addition of the original numbers!

cos(α + β) = cos(α)cos(β) − sin(α)sin(β)

If α = wk·a and β = wk·b (where wk = 2πk/P), then:

cos(wk·a)·cos(wk·b) − sin(wk·a)·sin(wk·b) = cos(wk·(a+b))

This is how the network computes a+b without ever "adding" directly! It multiplies the sine/cosine embeddings of a and b to get the sine/cosine of (a+b).

Abitur Connection: This is the Additionstheorem you learned! The network's MLP layer computes exactly this formula. The attention layer helps set up the products.

Above: The left circle shows the embeddings of a and b separately. The right circle shows the result — the point for (a+b) mod P. The network computed this by multiplying components, not by adding numbers!

Step 4: Constructive Interference — Reading Off the Answer

The final step: for each possible answer c (0 to 112), the network computes:

score(c) = Σk cos(wk·(a + b − c))

When c = a+b mod P, then a+b−c ≡ 0 mod P, so all cosines equal 1 and add up to a big peak. For wrong answers, the cosines point in different directions and mostly cancel out — destructive interference!

Abitur Connection: Think of wave superposition from physics! When waves are "in phase" they amplify (constructive interference). When they're out of phase, they cancel (destructive interference). Same principle here!

Step 5: The Grokking Phenomenon — Three Phases of Learning

Now the big picture: how does the network learn this algorithm? Not all at once! Training splits into three phases:

  1. Memorization (early): The network just memorizes training examples like a lookup table
  2. Circuit Formation (middle): The Fourier algorithm gradually forms in the weights, even while test accuracy stays low
  3. Cleanup (late): Weight decay removes the memorization, revealing the algorithm — test accuracy suddenly jumps!

Weight Decay: The Key Ingredient

Weight decay is a regularization technique that continuously pushes all weights toward zero. It's like a "simplicity pressure" — the network must justify every weight by how much it helps reduce loss.

The memorization solution needs many large weights (one for each training example). The Fourier algorithm needs fewer, more structured weights. So weight decay gradually favors the algorithm over memorization!

Summary of the Algorithm

1. Embed: Map a → (cos(wk·a), sin(wk·a)) for each key frequency k

2. Combine: Use trig identity to compute cos(wk·(a+b)) and sin(wk·(a+b))

3. Score: For each candidate answer c, compute Σk cos(wk·(a+b−c))

4. Output: The c with highest score wins — it's the correct answer!

Key Takeaways

🔑 Neural networks can discover elegant mathematical algorithms — this one found Fourier analysis on its own!

🔑 Grokking isn't sudden — the algorithm forms gradually, hidden beneath memorization

🔑 Regularization (weight decay) is crucial — it provides the pressure to prefer the simpler, generalizing solution

🔑 Mechanistic interpretability works — we can actually understand what networks learn, not just that they learn
