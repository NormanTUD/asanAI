The Big Picture: What Is Bayesian Optimization?

The core idea in 30 seconds

Imagine you have an extremely expensive function to evaluate — maybe training a neural network with certain hyperparameters takes 8 hours. You can't just try thousands of random settings. Bayesian Optimization (BO) is a strategy that intelligently picks the next point to evaluate so you find the optimum in as few tries as possible.

BoTorch does this in a loop with two main ingredients:

BoTorch's key innovation: It uses Sample Average Approximation (SAA) — fixing random samples once and then using fast deterministic optimizers (L-BFGS-B) instead of slow stochastic gradient descent. This is backed by theoretical convergence guarantees.

Step 1 The Gaussian Process Surrogate

Building a cheap-to-evaluate model of the expensive function

A Gaussian Process (GP) is a probabilistic model that, given some observed data points, provides a mean prediction (its best guess) and an uncertainty band (how confident it is) at every point in the search space.

Where we have data, the GP is confident. Where we don't, uncertainty is high. BoTorch uses GPyTorch under the hood, which provides fast O(1) predictive distributions.

Step 2 Monte Carlo Sampling & the Reparameterization Trick

How BoTorch makes acquisition functions differentiable

Most useful acquisition functions can't be computed in closed form. Instead, BoTorch approximates them using Monte Carlo samples from the GP posterior. The critical trick is the reparameterization trick:

Instead of sampling ξ ~ N(μ, σ²) directly, we write ξ = μ + σ · ε where ε ~ N(0,1). Now the sample is a differentiable function of the GP parameters μ and σ, which means we can compute gradients through the samples using auto-differentiation.

Step 3 The Acquisition Function

Scoring every candidate point: "How useful would it be to evaluate here?"

The acquisition function assigns a score to each candidate point. It balances exploration (try uncertain regions) vs. exploitation (try near the current best). BoTorch computes this as:

α(x) = (1/N) Σ a(g(ξi(x)))

where ξi are the fixed MC samples, g is an objective transform, and a is the utility (e.g., improvement over current best).

Step 4 Optimizing the Acquisition Function

Multi-start L-BFGS-B with smart initialization

The acquisition function surface is highly non-convex and multi-modal. BoTorch uses a two-phase strategy:

Phase 1 — Initialization: Generate thousands of quasi-random (Sobol) candidate points, batch-evaluate the acquisition function at all of them in parallel (exploiting GPU), then select the best ones as starting points.

Phase 2 — L-BFGS-B: From each starting point, run the L-BFGS-B quasi-Newton optimizer. Because the base samples are fixed (SAA), this is a smooth deterministic optimization. Pick the best result across all restarts.

Bonus SAA Convergence: MC vs Quasi-MC

Why fixing samples and using Sobol sequences works so well

BoTorch's theoretical contribution (Theorem 1 in the paper) proves that the SAA optimizer converges to the true optimizer exponentially fast. Quasi-Monte Carlo (Sobol) sequences converge even faster in practice than plain MC.
