Imagine you have a clock with P numbers on it (like a normal clock has 12). We want to teach a computer to add two numbers on this clock.

For example, on a clock with P=7: 5 + 4 = 2 (because 9 wraps around past 7).

The amazing discovery: the neural network figures out, all by itself, that it should put numbers on a circle and use rotation to add them!

We train a small transformer neural network to compute a + b mod P for a prime P (like P=113). The network discovers an algorithm that:

  1. Maps each number to points on circles (using sine and cosine)
  2. Combines the circle positions using trigonometric identities
  3. Reads off the answer by checking which output position has the strongest signal

This is remarkable because nobody told the network to use circles or trig — it invented this approach!

Step 1: Numbers Live on Circles — Angles & Frequencies

Simple
More Detail
Math

Think of a clock. Each number gets a position on the circle. The "frequency" k tells you how many times you go around the circle as you count from 0 to P-1.

Frequency k=1: Numbers are evenly spaced, going around once.
Frequency k=2: Numbers go around twice (so they're more "squished").
Frequency k=3: Three times around, etc.

The network picks a few special frequencies (called "key frequencies") and uses those circles to do its computation.

For a prime P, we define the angular frequency:

Each number n ∈ {0, 1, ..., P-1} maps to an angle wk·n on the circle. Different values of k give different "views" of the numbers — like looking at the clock through different lenses.

The key insight: addition of numbers corresponds to addition of angles. If a maps to angle wk·a and b maps to angle wk·b, then a+b maps to angle wk·(a+b). This works mod P because the angles repeat every P steps!

Since wk = 2kπ/P, we have wk·P = 2kπ, which means cos(wk·(n+P)) = cos(wk·n). The function is periodic with period P — exactly what we need for modular arithmetic!

Step 2: Where Cosine and Sine Come From — The Circle Connection

Simple
More Detail
Math

Cosine and sine are just the x and y coordinates of a point moving around a circle!

As you walk around a circle of radius 1:
• Your x-position (left-right) = cosine of the angle
• Your y-position (up-down) = sine of the angle

Watch the animation below — the blue dot goes around the circle, and its x-coordinate traces out the cosine wave, while its y-coordinate traces out the sine wave.

The unit circle (radius = 1, centered at origin) defines cosine and sine geometrically:

For any angle θ, the point on the unit circle at that angle has coordinates (cos θ, sin θ).

This means:
• cos²θ + sin²θ = 1 (always on the circle!)
• cos(0) = 1, sin(0) = 0 (start at the right)
• cos(π/2) = 0, sin(π/2) = 1 (top of circle)
• cos(π) = -1, sin(π) = 0 (left side)

The network needs to represent each number as a list of values (called an "embedding"). It learns to make these values follow sine and cosine patterns!

For each "key frequency" k, the number n gets mapped to two values: cos(wk·n) and sin(wk·n). This is like recording the x and y position on a circle.

Step 4: The Magic Trick — Adding Angles with Trigonometric Identities

Simple
More Detail
Math

Here's the key trick: if you know the circle-position of number a and the circle-position of number b, you can figure out the circle-position of a+b using just multiplication and subtraction!

It's like a recipe:
• To get the x-coordinate of (a+b): multiply the x's together, then subtract the product of the y's
• To get the y-coordinate of (a+b): cross-multiply (x of one times y of the other) and add

This is what the attention and MLP layers compute!

The network implements these formulas! The attention heads create products of sines and cosines (degree-2 polynomials), and the MLP combines them according to these identities.

Step 5: The Full Algorithm — From Input to Answer

Simple
More Detail
Math

The full algorithm in plain English:

  1. Embed: Put numbers a and b on circles (get their cos and sin at key frequencies)
  2. Combine: Use the trig trick to find cos and sin of (a+b) at each frequency
  3. Score each answer: For each possible answer c, compute cos(wk(a+b−c))
  4. Add up scores: Sum the scores from all frequencies. The correct answer gets the biggest total!

The correct answer c* = (a+b) mod P gets cos(0) = 1 from every frequency, so it wins!

This uses one more trig identity: cos(α−β) = cos α cos β + sin α sin β. The network already has cos(wk(a+b)) and sin(wk(a+b)) from Step 4. The unembedding matrix WL contains cos(wkc) and sin(wkc) for each output position c.

Step 6: Why Multiple Frequencies? — Constructive Interference

Simple
More Detail
Math

One cosine wave has a clear peak at the right answer, but it also has values close to 1 at other positions (false peaks). By adding multiple cosine waves at different frequencies, the peaks line up only at the correct answer and cancel out everywhere else!

It's like tuning multiple radio stations — they only agree at one "channel" (the right answer).

For a single frequency k, cos(wk·x) = 1 at x=0 but can be close to 1 at other values of x. For example, cos(2π·14/113 · 8) ≈ 0.998 — dangerously close!

But cos(2π·35/113 · 8) ≈ −0.990. Adding these two: 0.998 + (−0.990) ≈ 0.008, which is tiny! The false peak is destroyed.

With 5 frequencies, the sum at x=0 is 5 (all peaks align), but everywhere else the waves mostly cancel.

Step 7: The Real Network — Putting It All Together

Simple
More Detail
Math

The real trained network (P=113, 5 key frequencies k∈{14,35,41,42,52}) does exactly what we described! Here's the full pipeline running on real data:

  1. Input numbers a and b are embedded onto circles at 5 frequencies
  2. The attention + MLP layers compute cos(wk(a+b)) and sin(wk(a+b)) for each k
  3. The unembedding scores each possible answer c using cos(wk(a+b−c))
  4. The 5 cosine waves add up (constructive interference) to make the correct answer have the highest score

The real network explains 95% of variance in its logits with just these 5 cosine terms! [[11]]

From the paper (Table 1), the real network's MLP activations projected onto the unembedding directions give:

These are the actual measured coefficients from the trained network! The network independently discovered this algorithm through gradient descent. [[11]]

🎉 Summary: The Complete Algorithm

1. Embed

Map numbers to (cos, sin) pairs on circles at key frequencies

2. Combine

Use trig identities to compute cos/sin of the sum (a+b)

3. Score

For each candidate c, compute cos(wk(a+b−c))

4. Interfere

Sum across frequencies — correct answer wins by constructive interference!
