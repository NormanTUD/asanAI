Set Theory vs Homotopy Type Theory

Two competing foundations for all of mathematics — one classical and well-established, the other modern and computationally native. Explore how they differ in philosophy, structure, and expressiveness.

Set Theory (ZFC)

The classical foundation. Since the early 20th century, Zermelo-Fraenkel set theory with the Axiom of Choice has served as the standard foundation for mathematics.

Everything is a set. Numbers, functions, spaces — all are encoded as sets of sets. Mathematics is built on top of first-order logic, which provides connectives, quantifiers, and inference rules before any mathematical content begins.

Equality is simple: two sets are equal if and only if they have the same elements (Axiom of Extensionality).

Homotopy Type Theory (HoTT)

A modern, computational foundation. HoTT treats types as spaces and terms as points in those spaces. It exploits deep connections between homotopy theory and type theory.

Logic is not built separately — it emerges as a special case of the type-forming rules. Propositions are types, and proofs are terms inhabiting those types.

Equality is rich: the identity type between two terms forms a space of paths, which can themselves have paths between them (higher-dimensional structure).

Key Insight

In set theory, equality is a binary relation — either two things are equal or they aren't. In HoTT, the identity type Id(A, a, b) is itself a type that can have multiple inhabitants (multiple "proofs" or "paths" showing a = b). This captures the idea that there can be different ways in which two things are the same, which is fundamental in topology and category theory.

ZFC Axioms

  • Extensionality: Sets with same elements are equal
  • Pairing: For any a, b there exists {a, b}
  • Union: Union of a set of sets exists
  • Power Set: Set of all subsets exists
  • Infinity: An infinite set exists
  • Separation: Subsets defined by properties exist
  • Replacement: Images of sets under functions exist
  • Foundation: No infinite descending ∈-chains
  • Choice: Products of non-empty sets are non-empty

HoTT Rules & Axioms

  • Type Formation: Rules for forming Π, Σ, +, Id types
  • Introduction: How to construct terms of each type
  • Elimination: How to use/destruct terms
  • Computation: β-reduction and η-expansion
  • Universe Hierarchy: U₀ : U₁ : U₂ : ...
  • Univalence Axiom: (A ≃ B) ≃ (A =_U B)
  • Higher Inductive Types: Types with path constructors
  • Function Extensionality: Pointwise equal → equal

Structural Difference

Set theory states axioms within first-order logic — you need the entire logical apparatus before you can even state the axioms. Type theory's rules are the logic. The type-forming rules simultaneously define what objects exist and what you can prove about them. This makes type theory inherently computational: every proof corresponds to a program.

Paths as Equality Proofs

In HoTT, a proof that a = b is literally a path from point a to point b in the type (viewed as a space). A proof that two proofs of a = b are "the same" is a path between paths — a homotopy. This gives rise to an infinite tower of higher-dimensional structure, making types behave like ∞-groupoids.

The Circle as a Higher Inductive Type

In HoTT, you can define the circle S¹ with just: a point base : S¹ and a path loop : base = base. The fundamental group π₁(S¹) ≅ ℤ is then a theorem, not an axiom. This is impossible to express so directly in set theory.

Set Theory: Cumulative Hierarchy

Sets are built in stages: V₀ = ∅, V_{α+1} = P(V_α). The universe of all sets doesn't exist (Russell's paradox). We work "inside" the universe without being able to talk about it as an object.

HoTT: Universe Hierarchy

Types live in universes: U₀ : U₁ : U₂ : ... Each universe is a type in the next. The Univalence Axiom says: equivalent types in a universe are equal in that universe. This is the key innovation of HoTT.

Set Theory & Computation

-- In set theory, "2 + 2 = 4" is a theorem -- proved externally in first-order logic. -- There's no built-in notion of -- "computing" 2 + 2. -- Addition is defined as a set of triples: -- + = {(m,n,p) | p = m + n} -- where m + 0 = m -- m + S(n) = S(m + n) -- Verification requires logical deduction, -- not computation.

HoTT & Computation

-- In type theory, 2 + 2 COMPUTES to 4. -- This is called "definitional equality." -- nat : Type -- zero : nat -- succ : nat -> nat -- add : nat -> nat -> nat -- add m zero = m (β-rule) -- add m (succ n) = succ (add m n) -- add (succ(succ zero)) (succ(succ zero)) -- => succ (add (succ(succ zero)) (succ zero)) -- => succ (succ (add (succ(succ zero)) zero)) -- => succ (succ (succ (succ zero))) -- = 4 ✓ (by computation!)

Proofs as Programs (Curry-Howard)

In HoTT, every proof is a program and every program is a proof. The type of a program is its specification. Type-checking is proof-checking. This means mathematics done in HoTT is automatically verifiable by computer — tools like Coq, Agda, and Lean implement this directly.

Philosophical Differences

Summary: When to Use What?

Set Theory Strengths

• Well-understood, vast existing literature
• Flexible — anything can be encoded
• Good for classical analysis, combinatorics
• Independence results (forcing, large cardinals)
• Mature model theory

HoTT Strengths

• Natively computational (proofs = programs)
• Natural for homotopy theory & higher categories
• Machine-verifiable proofs
• Constructive by default
• Univalence captures mathematical practice
• Synthetic approach to topology
