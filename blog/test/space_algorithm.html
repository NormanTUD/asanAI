How Space Morphing Works

A visual, equation-by-equation walkthrough of the Metric Space Explorer algorithm

0

The Big Picture

A transformer has L layers. Each layer takes a token's hidden state and moves it — like a current pushing a particle through water. The Metric Space Explorer visualizes these movements as continuous deformations of space itself.

🔑 Core Insight

Instead of just tracking where individual tokens go, we ask: "If the entire space were made of rubber, how would each layer stretch, compress, and twist it?"

Each layer deforms the space — tokens ride the deformation like particles in a flow

Layer Deltas — Stacked Arrows

The delta (displacement) at each layer is the difference between consecutive hidden states:

How much layer ℓ "pushes" token i. This is the velocity vector for that layer.

Crucially, these deltas stack. Layer 0 moves the token to a new position. Layer 1 starts from that position and moves it further. Layer 2 starts from there, and so on. The total displacement is the sum of all deltas:

The final position is the initial embedding plus all layer deltas accumulated.

PCA Projection to 2D

Hidden states live in high-dimensional space (e.g., 768-D for GPT-2). We can't visualize that directly, so we use PCA to find the 2 directions of maximum variance:

Center the data by subtracting the mean of all hidden states.

Compute the covariance matrix from centered data.

Project each hidden state onto the top-2 eigenvectors to get 2D coordinates.
💡 Why PCA?

PCA preserves the directions where tokens differ the most. It's like finding the best camera angle to see the most spread in the data. The deltas are projected the same way, so directions are consistent.

RBF Interpolation — From Points to a Field

We know how each layer moves each token. But we want to know how it moves every point in space. This is where Radial Basis Function (RBF) interpolation comes in.

🎯 The Key Idea

Imagine dropping a stone in water at each token's position. The ripples (Gaussian bells) overlap. At any point in space, the velocity is a weighted blend of all token velocities, where closer tokens have more influence.

Step 1: For a grid point p, compute the Gaussian weight to each token:

The weight decays exponentially with squared distance. σ controls how far each token's influence reaches.

Step 2: The interpolated velocity at p is the normalized weighted sum of all token deltas:

Each token's delta contributes proportionally to its weight. This creates a smooth, continuous vector field.

Strain tells us how much the space is being distorted at each point. We compare the distance between adjacent grid points before and after deformation:

Ratio of deformed distance to original distance. Strain = 1 means no distortion.
