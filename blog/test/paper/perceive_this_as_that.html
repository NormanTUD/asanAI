The P-Morphism (Analogy Arrow)

The mapping h : PCam → PEye preserves structure ("no-blur" condition):

No-blur condition: Whenever ρ₁(w, ι) ≠ u, then ρ₂(h(w), h(ι)) = ρ₁(w, ι).
Defined values are preserved; only undefined values may change.

4. Types of Cognitive Transitions

Interpretive (PrcE)

Same environment, different connotations.
Example: Two people perceive the same bookstore differently.

Literal-Analogical (PrcI)

Same connotations, different environments.
Example: Camera → Eye (same functional properties, different parts).

Pure Improvement

Same E, same I, but ρ becomes more defined (u → t or u → f).
Example: Peeking at a magazine cover to check if it's the December issue.

6. Structural Alignment & Monotonicity

Good analogies preserve not just individual features but structural relationships between them.

Subjacency (on E)

x is subjacent to y means: every connotation x has, y also has.

Subsumption (on I)

ι subsumes κ means: every w-element with ι also has κ.

Interactive: Explore Structure in Camera & Eye

Monotone p-morphism: Preserves both subjacency and subsumption orderings. The camera-eye analogy is not fully monotone — it "slips" at the lens-film distance ↔ eye lens mapping, precisely where the analogy breaks down (different focusing mechanisms).

Unification: The same categorical primitives (E, I, ρ, p-morphisms) formalize:

2. Folds & Unfolds — The Building Blocks

Fold (Catamorphism)

Consumes a data structure by replacing constructors with operations.

Unfold (Anamorphism)

Produces a data structure from a seed value using a coalgebra.

Key Insight: A sorting algorithm goes from µList → νList̄ — consuming an unsorted list and producing a sorted one. This can be done as a fold of an unfold or an unfold of a fold.

3. The Swap — Computational Essence

Key Discovery: The "essence" of bubble sort and naïve insertion sort is a single natural transformation called swap — it swaps two adjacent elements if they're out of order.

The swap function (Haskell)

swap :: List (List̄ x) → List̄ (List x) swap Nil = Nil swap (Cons a Nil) = Cons a Nil swap (Cons a (Cons b x)) | a ≤ b = Cons a (Cons b x) -- already in order | otherwise = Cons b (Cons a x) -- swap!

The same swap gives rise to both bubble sort (unfold ∘ fold) and naïve insertion sort (fold ∘ unfold)

5. Para- & Apomorphisms — Efficiency Upgrade

Plain folds/unfolds can't stop early or peek at remaining input. We need richer schemes:

Paramorphism

Like a fold, but the algebra also sees the remainder of the input (like an as-pattern).

Used for: Selection sort (select the minimum while keeping the rest)

Apomorphism

Like an unfold, but can stop early — signal "I'm done, use this result directly".

Used for: Insertion sort (stop once element is in place)

Quicksort

Phase 1: Partition around pivots (unfold of fold)
Phase 2: Append partitions together (fold of apo)

quickSort = flatten · downcast · grow grow = unfold (para pivot) flatten = fold (apo glue)

Treesort

Phase 1: Insert elements into tree (fold of apo)
Phase 2: Extract elements in order (unfold of para)

treeSort = flatten' · downcast · grow' grow' = fold (apo treeIns) flatten' = unfold (para shear)
Natural transformation sprout captures the essence of both: pivot (for quicksort) and treeIns (for treesort) are derived from the same sprout!

Interactive: Heap Operations

The pile natural transformation

pile :: List (x × Heap x) → Heap (x + List x) pile Nil = ε pile (Cons a (t ε)) = (t▪) ⟨a⟩ (t▪) pile (Cons a (t (l ⟨b⟩ r))) | a ≤ b = (▸(Cons b r)) ⟨a⟩ (l▪) -- swap left/right | otherwise = (▸(Cons a r)) ⟨b⟩ (l▪) -- (Braun trick)

Key choice: Always add to right, then swap left↔right → builds a balanced Braun tree!

Heapsort

heapSort = unfold deleteMin · downcast · fold heapInsert

Build heap by insertion, extract by repeated deleteMin.

Minglesort (dual)

mingleSort = fold (apo blend) · downcast · unfold (fold divvy)

Divide list into balanced parts + min, then merge back. Like mergesort!

8. The Complete Picture

The Recipe:
  1. Choose an intermediate data structure (List, Tree, Heap)
  2. Define a natural transformation (swap, swop, sprout, pile, wither, sift)
  3. Derive algebra + coalgebra from it
  4. Get two dual sorting algorithms for free!
