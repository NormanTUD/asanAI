Geometric Structure of Layer Updates in Deep Language Models

An interactive visual exploration of the paper by Jun-Sik Yoo (2026)

What Does This Paper Study?

Deep language models (like GPT, Pythia, Mamba) transform token representations layer by layer. This paper asks: How do representations change from one layer to the next? Instead of asking "what information is stored," it asks about the geometry of the transformation itself.

The Core Idea

hℓ+1 = T(hℓ) + r(hℓ)
Each layer update is decomposed into:
• T(hℓ) — A "tokenwise" component (acts on each token independently)
• r(hℓ) — A "residual" component (everything else: cross-token interactions, etc.)
The full layer update is almost perfectly aligned with the tokenwise component. The residual is geometrically distinct — and it's where the functionally important computation lives.

Interactive Overview

The Decomposition: Tokenwise + Residual

The paper fits a local, input-conditioned tokenwise map to approximate each layer's transformation. For each token representation xi, the tokenwise map is:
T(xi) = A(xi) · xi
Where A(xi) is a matrix that depends on the input — making it globally nonlinear but locally linear. The residual is then: r = hℓ+1 - T(hℓ)

Geometric Structure: Angles and Alignment

The key geometric finding: the full update Δfull is nearly parallel to the tokenwise update Δtok, while the residual r points in a very different direction. This is measured using cosine similarity and angular deviation.

Alignment: Full Update vs. Tokenwise vs. Residual

The paper measures alignment using absolute cosine similarity: Align(v, u) = |v⊤u| / (‖v‖ · ‖u‖). This quantifies how much two vectors point in the same direction.

Functional Consequences: Residual Predicts Output Change

The paper's most striking finding: when the tokenwise approximation is poor (large residual), the model's output changes significantly. This is measured by KL divergence between original and perturbed output distributions, correlated with approximation error.

Subspace Projection: Where Does the Update Live?

The paper computes how much of each vector's energy falls within the top-k singular vectors of the local map. The full update lives in a low-dimensional subspace; the residual does not.
Projection fraction = ‖Uk⊤v‖² / ‖v‖²

Interactive Subspace Visualization

Even with just 1-8 singular vectors (out of hundreds of dimensions), the full and tokenwise updates are well-captured. The residual requires many more dimensions — it's spread across the space.

Architecture Comparison: Transformers vs. State-Space Models

A key contribution: these findings hold across architectures. Both Transformer models (Pythia, DistilGPT2) and state-space models (Mamba) show the same geometric separation — even though Mamba has no attention mechanism.

Cross-Architecture Comparison

Model Size Effects

The geometric structure is not specific to attention-based architectures. It emerges from the learned representation dynamics, not from architectural design choices.

Regimes & Trade-offs

The paper identifies distinct regimes: in low-error regimes, linear maps work great and show strong error↔output correlation. In high-error regimes, linear maps fail but MLPs partially recover. This reveals a fundamental trade-off between expressivity and interpretability.

Sensitivity Analysis

More expressive function classes reduce the residual but may weaken the interpretable relationship between residual magnitude and functional importance. The "sweet spot" depends on the regime.
