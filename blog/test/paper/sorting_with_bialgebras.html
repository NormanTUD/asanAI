1. The Big Picture

This paper reveals a hidden duality among sorting algorithms. The key insight: every sorting algorithm has a dual that you get "for free" by flipping the construction.

The paper distills each sorting algorithm down to a single natural transformation — a one-step operation that captures the "computational essence." From this, two dual algorithms emerge: one as a fold of unfold and one as an unfold of fold.

2. Folds & Unfolds

Fold (Catamorphism): Consumes a data structure by replacing constructors with operations.
Unfold (Anamorphism): Produces a data structure from a seed value.

Key property: The running time of a fold is proportional to the input size; the running time of an unfold is proportional to the output size. When we nest them (fold of unfold, or unfold of fold), we get quadratic time.

3. The Swap: Distributive Law

The swap function is the "computational essence" of bubble sort and naïve insertion sort. It compares two adjacent elements and swaps them if out of order.

Showing swap operations one at a time

As a distributive law: swap distributes the "head of a list" over the "head of a sorted list." It's natural in its type parameter — meaning it works uniformly regardless of what the rest of the list contains.

4. Bubble Sort ↔ Naïve Insertion Sort

Both algorithms use the same swap operation. The difference is how they're assembled:

Bubble Sort unfold(fold swap)

Repeatedly folds the list to bubble the minimum to the front, then unfolds by peeling off that minimum.

Naïve Insert Sort fold(unfold swap)

Folds over the input, at each step unfolding to insert the element into the correct position in the sorted output.

The Duality: By the theory of bialgebras, bubbleSort = naïveInsertSort. They are the unique arrow from the initial to the final swap-bialgebra. Two descriptions of the same computation!

5. Paramorphisms & Apomorphisms

To get efficient sorting, we need recursion schemes that can "see more" or "stop early."

Paramorphism enhanced fold

Like a fold, but the algebra also sees the remainder of the input (not just the accumulated result). Think: Haskell's @-pattern.

Apomorphism enhanced unfold

Like an unfold, but the coalgebra can signal STOP — "I'm done, use this value directly without further recursion."

6. Insertion Sort ↔ Selection Sort

Using swop (swap + stop), we get the true insertion and selection sorts. These are more efficient than the naïve versions because they can terminate early.

Insertion Sort fold(apo swop)

Folds over input; each step uses an apomorphism to insert into sorted list. Stops early once element is placed. Best case: O(n).

Selection Sort unfold(para swop)

Unfolds output; each step uses a paramorphism to select minimum. Uses remainder of input (not just result so far). Always O(n²).

7. Quicksort ↔ Treesort

To break the O(n²) barrier, we introduce an intermediate data structure: binary search trees. The algorithm becomes two phases: build the tree, then flatten it.

Quicksort

Phase 1 (grow): Recursively partition list around pivot → builds tree top-down
Phase 2 (flatten): Append sublists together (in-order traversal)

Treesort

Phase 1 (grow'): Insert elements one-by-one into search tree → builds tree bottom-up
Phase 2 (flatten'): Extract elements in order

Interactive: Watch Tree Construction

Both produce the same search tree — just built differently!

Natural transformation sprout: Maintains the search tree property when inserting. Quicksort's pivot and treesort's treeIns are both derived from it — dual to each other. Average case: O(n log n). Worst case: O(n²) due to unbalanced trees.

8. Heapsort & Minglesort

To guarantee O(n log n), we use a heap (vertically ordered tree) instead of a search tree. The heap's partial ordering gives flexibility in arrangement, avoiding worst-case unbalanced trees.

µList (input)
→
Heap (Braun tree)
→
νList (sorted output)

Heapsort

Phase 1: Build heap by repeated insertion (fold of apo)
Phase 2: Repeatedly extract minimum (unfold of para = deleteMin)

Minglesort dual

Phase 1: Build heap by repeated selection of minimum (unfold of fold = divvy)
Phase 2: Merge sublists together (fold of apo = blend)

9. Summary: The Sorting Zoo

Every sorting algorithm in this paper is characterized by a natural transformation (the one-step essence) and a recursion scheme (how to apply it repeatedly). Duality gives us algorithms for free!

Key Takeaways

  • Distributive law (swap/swop/sprout/pile/wither/sift): The one-step computational essence
  • Bialgebra: An algebra + coalgebra on the same carrier, related by the distributive law
  • Initial bialgebra → unfold of fold; Final bialgebra → fold of unfold
  • Duality: Fold ↔ Unfold, Paramorphism ↔ Apomorphism, Insertion ↔ Selection
  • Intermediate structures: Lists → Trees → Heaps give progressively better guarantees
