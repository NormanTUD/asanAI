How Neural Networks Decode Superposition in Deep Layers

An interactive exploration of how internal layers use topological structures, Fourier decomposition, and constructive interference to transform superimposed representations into meaningful computations.

What is Superposition in Neural Networks?

In deeply nested networks, neurons encode multiple features simultaneously in the same set of weights. This is called superposition - the network stores more concepts than it has dimensions by overlapping them in a high-dimensional space.

Key Insight: The embedding matrix maps discrete inputs onto continuous rotations in high-dimensional space. For modular addition, inputs a and b are mapped to sin(w_k * a), cos(w_k * a), sin(w_k * b), cos(w_k * b) at specific "key frequencies" w_k.

Topology Connection: The circle (S1) is the fundamental topological space here. The embedding creates a homomorphism from the discrete group Z/PZ to rotations on the circle. Multiple frequencies create a product of circles (a torus), giving the network a rich topological manifold to work with.

Fourier Topology: How Layers Organize Information

The network's internal representations are sparse in the Fourier basis. Out of all possible frequencies, only a handful of "key frequencies" carry meaningful information. This creates a topological structure where computations are localized by frequency.

From the Research: In a trained transformer doing modular addition (mod 113), the embedding matrix W_E is sparse in the Fourier basis - only 5 key frequencies (k = 14, 35, 41, 42, 52) have significant norm. Each neuron operates primarily on a single frequency, creating frequency-localized computation.

Layer-by-Layer Computation: From Superposition to Answer

Each layer performs a specific topological operation. The attention layer computes products of trigonometric functions (bilinear mixing), and the MLP layer uses trigonometric identities to compose rotations.

The Algorithm:
1. Embed: Map a, b to sin(w_k*a), cos(w_k*a), sin(w_k*b), cos(w_k*b)
2. Attention + MLP: Compute cos(w_k*(a+b)) and sin(w_k*(a+b)) via trig identities
3. Unembed: For each output c, compute cos(w_k*(a+b-c)) and sum over frequencies
4. Interference: Constructive interference at c* = a+b mod P gives the answer

84.6% of MLP neurons are well-approximated by degree-2 polynomials of a single frequency. Each neuron "specializes" in one frequency.

Constructive Interference: How the Answer Emerges

The final decoding step uses constructive interference - multiple cosine waves at different frequencies all peak at the correct answer c* = a+b mod P, while destructively interfering everywhere else. This is the topological "readout" mechanism.

Why Multiple Frequencies? A single cosine cos(2pi*k*x/P) has its maximum at x=0, but other values of x can be nearly as large. By summing cosines at multiple key frequencies, the network creates a function where ONLY x=0 mod P gives a large value - all other peaks cancel out through destructive interference.

Removing key frequencies destroys performance. Removing all OTHER frequencies actually improves it!

Grokking: The Three Phases of Learning

Grokking is the phenomenon where a network suddenly generalizes long after memorizing training data. Using progress measures from mechanistic interpretability, we can see that this "sudden" shift is actually the result of three continuous phases.

The Three Phases:
1. Memorization (0-1.4k epochs): Network memorizes training data. Key frequencies unused.
2. Circuit Formation (1.4k-9.4k epochs): Fourier multiplication circuit gradually forms. Weight decay drives this.
3. Cleanup (9.4k-14k epochs): Weight decay removes memorization components. Test accuracy suddenly jumps.
