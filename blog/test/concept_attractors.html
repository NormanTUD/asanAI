The Core Insight

When you ask an LLM about "Who is Gandalf the Grey?" and "What is the significance of Mount Doom?" — these prompts look completely different on the surface. Yet inside the model, at a specific layer, their internal representations converge to nearly the same point.

This paper shows that this convergence isn't random — it's the behavior of a dynamical system with concept-specific Attractors. And once you understand this, you can manipulate these Attractors to do remarkable things without any training.

What Are Attractors?

In dynamical systems, an Attractor is a set of states toward which a system tends to evolve, regardless of where it starts. Think of it like a drain in a bathtub — no matter where you place the water, it flows toward the drain.

Types of Attractors

Attractors come in different forms depending on the system:

⚫
Fixed Point

The simplest attractor. All trajectories converge to a single point. Like a ball rolling to the bottom of a bowl.

🔄
Limit Cycle

Trajectories converge to a repeating loop. Like a heartbeat or a pendulum with friction.

🌊
Strange Attractor

Complex, fractal-like structures. Trajectories are attracted but never repeat exactly. The Lorenz attractor is famous.

In this paper, the authors discover that LLM layers act as contractive mappings that push representations of related concepts toward concept-specific attractors. Different concepts converge at different layers — Lord of the Rings at layer 24, programming languages at layer 19, literature at layer 18.

Iterated Function Systems (IFS)

An Iterated Function System is a collection of contractive mappings. When you repeatedly apply these mappings to any starting set, it converges to a unique invariant set — the Attractor. This is exactly what the paper claims LLM layers do!

F(S) = ⋃ᵢ fᵢ(S) → The Hutchinson operator combines all contractive maps
F(F(F(...F(S₀)...))) → S* → Repeated application converges to the Attractor S*

The Collage Theorem

The Collage Theorem is key to the paper's methodology. It states: if you can find contractive maps that "tile" (collage) a target set well, then the attractor of those maps will be close to that target. The authors use this to find the effective contractive map that each LLM implements for each concept.

min_{M_eff, t_eff, iter} Σⱼ D(h_l(pⱼ), φ^iter_eff(h₀(pⱼ)))
subject to: ||M_eff||_op < 1 (contractivity constraint)

Where φ_eff(V) = M_eff · V + t_eff is the effective affine contractive map

Application: Concept Detection & Guardrailing

Since concepts form distinct Attractors, we can detect when a prompt belongs to a specific concept by measuring how close its representation is to the known Attractor. This enables training-free "unlearning" — blocking responses about specific topics.

Attractor Traversals: Steering the LLM

If the LLM is a dynamical system, we can intervene on its trajectory and guide it toward a different Attractor. The modification is elegantly simple:

dx/dt = f(x) + λ(y − x)

Where f(x) is the original dynamics, y is the target Attractor,
and λ controls the steering strength.

In practice, this means adding or subtracting Attractor vectors at specific intermediate layers using a simple forward hook. No retraining, no fine-tuning, no retention data.

Code Translation (Transpiling)

Programming languages form Attractors at layer 19. Switch from one language's Attractor to another to transpile code — zero-shot, no instructions needed.

Input (Python):

def fib(n): return n if n<=1 else fib(n-1)+fib(n-2)

Output (JavaScript):

function fib(n) { return n<=1 ? n : fib(n-1)+fib(n-2); }

Visual Hallucination Reduction

In Vision-Language Models (VLMs), the model starts aligned with a visual Attractor but drifts toward a text-only Attractor during generation. The fix: continuously re-add the initial visual Attractor vector at each generation step to keep the model grounded in what it actually sees.

Attractor Perturbation for Data Generation

Temperature sampling adds randomness but often produces low-quality or repetitive outputs. The Attractor perspective offers a better approach: perturb the Attractor itself to generate diverse, high-quality samples from a single prompt.

Interactive: Temperature vs. Attractor Perturbation

Left: Temperature sampling scatters outputs randomly (many fall outside the "quality zone"). Right: Attractor perturbation explores the space around the Attractor systematically, staying within the quality zone. Click to generate samples.

Interactive

Key Insight: Sample-Wise Attractors

Different generation instructions for the same sample converge to the same point at layer 16. This means we can replicate the diversity of multiple carefully-crafted prompts by simply perturbing a single Attractor — no prompt engineering needed.

Multiple instructions → Same Attractor (layer 16) for same sample
∴ Perturbing Attractor ≈ Using multiple diverse instructions

Result: 20%+ factuality improvement, better downstream task accuracy

The Fractal Structure

Attractors aren't just single points — they have fractal-like hierarchical structure. For arithmetic tasks, the Attractor splits by number of digits, then by operation type, then by specific values. Using the wrong "level" of Attractor (e.g., averaging all digit-lengths together) produces a noisy, less effective representation.
