🧠 What is a Neural Network?

A neural network is simply a mathematical function that maps inputs to outputs. The "magic" is that it can represent an incredibly rich family of functions, and we can search through this family to find one that fits our data.

The Simplest Network: One Input, One Output

A shallow neural network with ReLU activations creates a piecewise linear function — essentially a series of connected line segments. Each hidden unit contributes one "joint" where the function changes slope.

Key Insight: With enough hidden units, a shallow network can approximate any continuous function. This is the Universal Approximation Theorem. More units = more joints = better approximation. But "enough" might be impractically many — which is why we use deep networks.

⚡ The ReLU: Simplicity is Power

The Rectified Linear Unit is embarrassingly simple: it returns its input if positive, and zero otherwise. Yet this tiny nonlinearity is what gives neural networks their power.

Why ReLU works: Its derivative is always 0 or 1. This means gradients flow cleanly during backpropagation — no vanishing gradients from saturating regions like sigmoid has. It's also computationally trivial: just a max(0, x) operation.

🏗️ Why Go Deep?

Deep networks compose simple functions to build complex ones. Each layer "folds" the input space, creating exponentially more linear regions than a shallow network with the same number of parameters.

The Folding Metaphor: Think of the first layer as "folding" the input space back on itself. The second layer then applies a function that gets duplicated at every fold. With K folds, you get exponentially many copies — this is depth efficiency.

🎯 Training: Walking Downhill

Training a neural network means finding parameters that minimize a loss function. We do this by iteratively computing the gradient (which direction is "downhill") and taking a step in that direction.

The learning rate dilemma: Too small → painfully slow convergence. Too large → overshooting and instability. Modern optimizers like Adam adapt the learning rate per-parameter, making training much more robust.

🔄 Backpropagation: The Chain Rule at Scale

Computing gradients for millions of parameters sounds impossible. Backpropagation makes it elegant: it's just the chain rule applied systematically, reusing computations as we work backward through the network.

Watch Signals Flow

Click "Forward" to see data flow through the network, then "Backward" to see gradients propagate back.

Two key observations: (1) Each weight's effect is scaled by the activation of its source neuron — so we store activations in the forward pass. (2) Changes ripple through subsequent layers — but we can reuse these calculations by working backward. This makes backprop linear in network size!

📊 Loss Functions: Measuring "Wrong"

The loss function quantifies how badly the model predicts the training data. Different tasks need different losses — and they all come from a beautiful unifying principle: maximum likelihood.

From Probability to Loss

Regression: Least Squares

Assumes Gaussian noise → minimizing squared errors

Classification: Cross-Entropy

Assumes Bernoulli distribution → binary cross-entropy

The recipe: (1) Choose a probability distribution over your output domain. (2) Use the network to predict its parameters. (3) Minimize the negative log-likelihood. That's it! Least squares, cross-entropy, and exotic losses all emerge from this one recipe.

🎭 The Generalization Mystery

A network can perfectly memorize training data yet fail on new examples. Understanding why models generalize (or don't) is one of the deepest open questions in deep learning.

Bias-Variance Trade-off & Double Descent

Double Descent: Classical theory says more parameters → more overfitting. But modern networks show a U-shaped test error that decreases again past the interpolation threshold! Over-parameterized models interpolate smoothly between data points, which generalizes well. Nobody fully understands why.

🔮 Transformers: Attention is All You Need

The transformer revolutionized AI by letting every element in a sequence attend to every other element. The key innovation: self-attention — a mechanism where the connections between words depend on the words themselves.

Self-Attention Visualized

Hover over words to see which other words they "attend" to. The thickness of connections shows attention strength.

Why attention works: Unlike RNNs that process words sequentially (and forget distant ones), transformers let every word directly interact with every other word. The attention weights are computed from the data itself — creating a dynamic, content-dependent wiring pattern. This is why "it" can attend to "restaurant" even many words apart.

🌌 Why Does Deep Learning Work?

Modern networks have more parameters than training examples, learn piecewise linear functions with more regions than atoms in the universe, yet generalize beautifully. This shouldn't work — but it does.

Open Mysteries

Loss Landscape: Despite having billions of parameters, the loss surface is surprisingly benign — most local minima are roughly as good as the global minimum.
Implicit Regularization: SGD doesn't just find any minimum — it's biased toward flat, wide minima that generalize better. The algorithm itself is a regularizer.
Lottery Tickets: Inside every large network is a small subnetwork that could have been trained in isolation to achieve similar performance.
Grokking: Networks can suddenly generalize long after perfectly memorizing training data — understanding emerges from continued training.
