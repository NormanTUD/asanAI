Homotopy Type Theory

A visual explanation of why types-as-spaces, equality-as-paths, and univalence give us a foundation for mathematics that actually makes sense — where structure is respected, proofs carry information, and equivalent things are truly equal.

Section 1

Types Are Not Just Sets

In set theory, a set is a bag of elements — {true, false} is Bool. But sets have no structure beyond membership. In HoTT, a type is a space with shape. The type Bool has exactly two points and no paths between them (true ≠ false, period). But a type like the Circle has one point and infinitely many loops. The shape of the space IS the mathematics.

❌ Set Theory

A set is just a bag. {a, b} has two elements. That's ALL you know. No notion of "how" elements relate. Equality is just "same element or not" — a single bit of information.

✓ HoTT

A type is a space. It has points (elements), paths (equalities), paths-between-paths, and so on. The topology of the type carries mathematical meaning. Bool is discrete — two isolated points. ℕ is a chain. The Circle has a loop.

Section 2

Equality Is a Path (Not a Bit)

Consider the expressions x + 1 + 1 and x + 2. They compute to the same thing. In classical math, we just say "they're equal" — a yes/no. But HOW do we know they're equal? There's a computation that transforms one into the other. In HoTT, that computation IS the equality — it's a path from one to the other. Different computations = different paths = different reasons why things are equal.

Watch: multiple different proofs that two expressions are equal — each is a distinct path

Why this matters: In set theory, "2+2=4" is just TRUE. In HoTT, the proof that 2+2=4 is a mathematical object — a path you can study, compose with other paths, or invert. Proofs aren't just certificates; they're first-class mathematical structures.
Section 3

Paths Between Paths — Infinite Depth

If two proofs of equality are themselves "the same proof," that sameness is another path — a path between paths. This is a homotopy: a continuous deformation of one path into another. And you can ask if two homotopies are the same... giving paths between paths between paths. This infinite tower of structure is exactly what topology studies — and HoTT makes it the foundation of all math.

The blue and red paths are two different proofs of A=B. The purple surface is a proof that those proofs are equal.

❌ Classical Logic

Two proofs of the same theorem? Irrelevant — a theorem is just "true." All proofs are interchangeable. We lose information.

✓ HoTT

Different proofs are different paths. Some can be deformed into each other (homotopic), some can't. This proof-relevant structure captures real mathematical content that classical foundations throw away.

Section 4

The Circle — Where Types Get Interesting

The circle S¹ is a type with one point (called "base") and one non-trivial path (a loop from base back to base). But you can go around the loop multiple times! Once, twice, backwards... The "equality type" base=base turns out to be the integers ℤ — the winding number. This is a type whose equality structure is infinitely rich, emerging purely from its shape.

The punchline: The integers aren't "defined" here — they emerge from the topology of the circle. In HoTT, you can define the circle as a type (one point + one loop), and the integers fall out as its fundamental group. Topology and algebra are unified at the foundational level.
Section 5

Univalence — The Killer Feature

Mathematicians constantly say things like "ℤ/2 and {even, odd} are the same thing." In set theory, this is technically false — they're different sets that happen to be isomorphic. You need to manually transport theorems between them. In HoTT, the Univalence Axiom says: equivalent types ARE equal. Period. If there's a perfect two-way mapping, they're literally the same type. Math finally works the way mathematicians actually think.

Watch: two equivalent types are revealed to be the SAME type — not just "similar"

❌ Set Theory (ZFC)

ℤ/2 = {0,1} with addition mod 2.
{even,odd} = a different set.
They're isomorphic but NOT equal. Every theorem about one must be manually "transported" to the other. Tedious. Error-prone.

✓ HoTT + Univalence

If A ≃ B (equivalent), then A = B (equal). Automatically. Any theorem about A is instantly a theorem about B. The foundation respects the way mathematicians actually work. No more "abuse of notation" — it's just correct.

Section 6

Seeing It Work: 2 + 2 = 4 as a Path

Natural numbers form a type — a chain of points connected by the successor function S. The number 2 is S(S(0)): start at 0, step twice. Addition "a + b" means: start at a, take b successor steps. So 2+2 means "start at 2, step twice" → land on 4. The proof of equality is a path from the expression "2+2" to the point "4" — and that path is refl (reflexivity), because after computation they're definitionally the same point.
