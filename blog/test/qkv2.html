Scene 1: What Attention Is

Each token is a point in embedding space. Attention moves each point toward a weighted average of other points — but in a different subspace than the one used to decide the weights.

Attention does NOT lay a surface or a simplicial complex. It computes a convex combination of value vectors. The output for each token is a point inside the convex hull of the value vectors, with position determined by query-key similarity.
Attention(Q, K, V) = softmax(QKᵀ / √d_k) · V
The geometry: You have n points (tokens) in ℝᵈ. Each point gets projected into three different subspaces via learned linear maps W_Q, W_K, W_V. The Q-K dot product measures angular alignment in the query-key subspace. This produces a probability distribution (via softmax). Then each token's new representation is the weighted centroid of all value vectors under that distribution. The output lives in the convex hull of the V vectors.

Scene 2: Q·K = Cosine Similarity (Scaled)

Click any token to be the "query." Watch how the dot product with every key determines alignment. Parallel = high score. Orthogonal = zero. Opposing = negative.

The Q and K projections define two different "views" of the same point. W_Q asks "what am I looking for?" and W_K asks "what do I advertise?" The dot product q·k measures how well one token's question matches another token's advertisement — purely by the angle between them (scaled by magnitudes).
score(i,j) = q_i · k_j / √d_k = |q_i||k_j|cos(θ_ij) / √d_k
Why √d_k? If q and k are random vectors in ℝᵈ with unit-variance components, their dot product has variance d_k. Dividing by √d_k normalizes this so softmax doesn't saturate. This is just a geometric rescaling to keep angles meaningful regardless of dimension.

Scene 3: Softmax Maps Scores onto the Probability Simplex

The raw scores are real numbers. Softmax maps them to a point on the (n-1)-simplex: all weights are positive and sum to 1. This is the "attention pattern."

Softmax doesn't just normalize — it exponentiates first, which amplifies differences. A score of 3 vs 1 becomes e³:e¹ ≈ 20:2.7. This makes attention either spread out (when scores are similar) or concentrate (when one score dominates). The temperature is controlled by 1/√d_k.
softmax(z_i) = exp(z_i) / Σ_j exp(z_j)
Result: a point on the (n-1)-simplex Δⁿ⁻¹ = {w ∈ ℝⁿ : w_i ≥ 0, Σw_i = 1}

Scene 4: Values = The Payload. Weights Choose the Mixture.

Q and K decide WHO to listen to. V decides WHAT they say. The output is a weighted centroid in value-space. Drag the attention weights to see the output move inside the convex hull.

The output for token i is: out_i = Σ_j α_ij · v_j. This is literally a weighted average — a point inside the convex hull of {v_1, ..., v_n}. When attention is sharp (one weight ≈ 1), the output copies one value vector. When diffuse, it's the centroid. Attention interpolates between copying and averaging.
output_i = Σ_j softmax(q_i · k_j / √d_k) · v_j
The full picture: Each token starts as a point x in ℝᵈ. Three linear maps (W_Q, W_K, W_V) project it to three different subspaces. Q and K live in ℝ^{d_k} where alignment is measured. V lives in ℝ^{d_v} where the actual information flows. The output is a convex combination of V vectors, with coefficients determined by Q-K geometry. Then W_O projects back to ℝᵈ. The whole thing is a learned, data-dependent, soft routing of information between positions.
Summary of the geometry:
• Q, K subspace: Where "who attends to whom" is decided via angular similarity
• Softmax: Maps ℝⁿ → Δⁿ⁻¹ (the probability simplex)
• V subspace: Where actual content lives
• Output: A point in the convex hull of V vectors (weighted centroid)
• Multi-head: Multiple independent (Q,K,V) triples in different subspaces, concatenated — lets the model attend to different things simultaneously
