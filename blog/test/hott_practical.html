What is Homotopy Type Theory?

In HoTT, types are spaces, terms are points, and equalities are paths. Instead of building mathematics on sets and membership, we build it on types and inhabitants. Every mathematical object is a type, and proving something means constructing a term of that type.

The Identity Type (Path Type)

For any type A and terms a, b : A, the identity type a =_A b represents the space of paths from a to b. This is the revolutionary idea: equality is not a proposition but a type that can have multiple inhabitants (multiple different paths/proofs of equality).

-- The identity type / path type
data _==_ {A : Type} (a : A) : A -> Type where
  refl : a == a

-- Path induction (J-eliminator)
J : (A : Type) -> (C : (x y : A) -> x == y -> Type)
  -> ((x : A) -> C x x refl)
  -> (x y : A) -> (p : x == y) -> C x y p

Constructing Natural Numbers (N)

Natural numbers are the first inductive type we build. In HoTT, N is defined by two constructors and an elimination principle:

1
Zero constructor: 0 : N
We have a base point - the number zero.
2
Successor constructor: S : N → N
Given any natural number n, we can form its successor S(n).
3
Recursion principle: To define a function out of N, provide a base case and a step.
This is what lets us define addition, multiplication, etc.
3

Addition on N via Recursion

-- Addition defined by recursion on the second argument
add : N -> N -> N
add m 0 = m
add m (S n) = S (add m n)

-- Example: 2 + 3
-- add 2 3 = add 2 (S 2) = S(add 2 2) = S(S(add 2 1))
-- = S(S(S(add 2 0))) = S(S(S(2))) = 5

Each step of the recursion corresponds to "walking one step further" along the natural number line. The path from the input to the output is the computation itself, witnessed by definitional equalities (refl).

Constructing Integers (Z)

Integers extend naturals with negatives. In HoTT, we can construct Z in multiple equivalent ways:

1
As a coproduct: Z := N + 1 + N
Negative naturals + zero + positive naturals. (N ⊔ {0} ⊔ N)
2
As a quotient: Z := (N × N) / ~
Pairs (a,b) representing a - b, quotiented by (a,b) ~ (c,d) iff a+d = b+c
3
As a loop space: Z ≃ Ω(S¹)
The integers are the loop space of the circle! This is a deep HoTT result.

Z ≃ Ω(S¹): Integers as Loops

One of the most beautiful results in HoTT: the fundamental group of the circle is Z. Each integer n corresponds to winding around the circle n times (negative = opposite direction). This gives integers a geometric meaning!

-- The circle type
data S¹ : Type where
  base : S¹
  loop : base == base

-- The fundamental theorem: Ω(S¹) ≃ Z
-- loop^n corresponds to integer n
encode : base == base -> Z
decode : Z -> base == base

Constructing Rationals (Q)

Rationals are constructed as a set-quotient of pairs of integers:

1
Raw fractions: Z × Z⁺
Pairs (numerator, positive denominator)
2
Equivalence relation: (a,b) ~ (c,d) iff a*d = c*b
Cross-multiplication identifies equivalent fractions
3
Set-quotient: Q := (Z × Z⁺) / ~
In HoTT, this is a higher inductive type (HIT) with path constructors for the equivalence

Division as a Type-Theoretic Operation

In HoTT, division on rationals is defined via multiplication by the inverse. The key insight is that the quotient type ensures well-definedness: we must prove that our operations respect the equivalence relation.

-- Rational division (multiply by inverse)
div : Q -> Q -> Q
div [a,b] [c,d] = [a*d, b*c] -- when c != 0

-- Well-definedness proof required:
div-resp : (a,b) ~ (a',b') -> (c,d) ~ (c',d')
  -> div [a,b] [c,d] ~ div [a',b'] [c',d']

Constructing Real Numbers (R)

The reals are where HoTT truly shines. There are multiple constructions, each revealing different aspects:

1
Cauchy Reals: R_C := CauchySeq(Q) / ~
Equivalence classes of Cauchy sequences of rationals. A sequence (q_n) is Cauchy if for all epsilon > 0, there exists N such that |q_m - q_n| < epsilon for m,n > N.
2
Dedekind Reals: R_D := {(L,U) : P(Q) × P(Q) | cuts}
A pair of subsets of Q (lower and upper) satisfying: inhabited, rounded, disjoint, located. Each cut "pinches" a real number from both sides.
3
HIT Reals (Cauchy completion): R := HIT with lim constructor
A higher inductive type with: rational embeddings, a limit constructor for Cauchy sequences, and path constructors ensuring limits of equivalent sequences are equal.

Univalence and Transport

The Univalence Axiom states that equivalent types are equal: (A ≃ B) ≃ (A =_U B). This means we can transport structure along equivalences. For arithmetic, this means:

  • Different constructions of Z (coproduct vs quotient vs loop space) are equal as types
  • Arithmetic defined on one construction automatically transfers to another
  • The path witnessing this equality IS the translation function
