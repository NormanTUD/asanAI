The Embedding Matrix WE: Where Sine is STORED

The embedding matrix is a d x P matrix (128 x 113 in the paper). Each column maps a one-hot token to a 128-dimensional vector. After training, these columns become sparse in the Fourier basis - they encode sin(wk * a) and cos(wk * a) for a handful of "key frequencies" k.

Answer to "where is sine stored?": The sine and cosine values are literally the learned weights of the embedding matrix. When you look at WE in the Fourier basis, you see that it is sparse - only 5 key frequencies have significant weight. The model learned to represent each number as a point on circles at these frequencies.

Embedding of each token a in 2D (cos(wk*a), sin(wk*a))

Fourier spectrum of embedding (only key freqs have large norm)

WE[token=a] ≈ Σk∈keys [ αk cos(2πk·a/P) + βk sin(2πk·a/P) ] · directionk

Attention Heads: The Multiplier

The attention mechanism is approximately bilinear - it multiplies things together. This is crucial because the trig identity for cos(a+b) requires products of sin and cos terms.

How attention helps: Each attention head's pattern is periodic in a single frequency. The attention score from "=" to token "a" is approximately:
0.5 + α(cos(wk·a) - cos(wk·b)) + β(sin(wk·a) - sin(wk·b))

The OV circuit (WOWV) then outputs the embedding weighted by this attention, creating products of sines and cosines - exactly what the trig identity needs!
Attention creates: cos(wk·a)·cos(wk·b), sin(wk·a)·sin(wk·b), sin(wk·a)·cos(wk·b), cos(wk·a)·sin(wk·b)

Two heads (0 and 2) compute degree-2 polynomials of a single frequency. The other two heads (1 and 3) amplify the key frequency embeddings in the residual stream.

The MLP (FFN): Where the Trig Identity is COMPUTED

The MLP layer takes the products created by attention and combines them using the trigonometric addition identities. Each neuron approximately computes a degree-2 polynomial of sines and cosines at a single frequency.

The key computation happens here! The MLP neurons compute:
cos(wk(a+b)) = cos(wk·a)cos(wk·b) - sin(wk·a)sin(wk·b)
sin(wk(a+b)) = sin(wk·a)cos(wk·b) + cos(wk·a)sin(wk·b)

84.6% of neurons (433/512) have over 85% of their variance explained by a degree-2 polynomial of a single frequency!

The Unembedding (WL = WUWout): Reading Off the Answer

The neuron-logit map WL is approximately rank 10 (5 frequencies x 2 for sin/cos). It "reads off" cos(wk(a+b)) and sin(wk(a+b)) from the MLP activations and multiplies them by cos(wk·c) and sin(wk·c) to produce cos(wk(a+b-c)).

The unembedding implements:
Logit(c) ≈ Σk αk · cos(wk(a+b-c))

This is maximized when c = a+b mod P (because cos(0) = 1 is the maximum). The different frequencies constructively interfere at the correct answer and destructively interfere elsewhere.

Logit for each possible output c. The peak is at c = a+b mod P.

Individual frequency contributions (colored) and their sum (white). Constructive interference at the correct answer!

The Final Softmax: From Logits to Prediction

After the logits are computed, softmax converts them to a probability distribution. Because the constructive interference creates a sharp peak at c* = a+b mod P, the softmax assigns nearly all probability to the correct answer.

Raw logits (before softmax)

After softmax (probability distribution)

Why softmax works so well here: The logits have a single sharp peak due to constructive interference of 5 cosine waves. Softmax exponentiates these values, making the peak even sharper. With trained models, the probability at the correct answer is essentially 1.0.

Why a Circle? The Modular Addition = Rotation Insight

The Core Idea

Numbers mod P naturally live on a circle! If you map number a to angle wk * a where wk = 2πk/P, then:

  • Addition mod P becomes rotation on the circle
  • The angle for (a+b) mod P is just the angle for a plus the angle for b
  • This works because eiw(a+b) = eiwa · eiwb
Why multiple frequencies? A single cosine cos(wk(a+b-c)) is maximized at c = a+b mod P, but it also has values close to 1 at other points. By summing 5 different frequencies, the model creates constructive interference ONLY at the correct answer, and destructive interference everywhere else.
cos(w14·x) + cos(w35·x) + cos(w41·x) + cos(w42·x) + cos(w52·x)
≈ 5 when x=0, ≈ 0 elsewhere

Summary: The Complete Zusammenspiel

Component What it stores/computes Role in the algorithm
Embedding WE sin(wk·a), cos(wk·a) for key frequencies STORES the Fourier representation of each number
Attention Heads Periodic patterns + bilinear products MULTIPLIES sin/cos terms together (needed for trig identity)
MLP/FFN Neurons cos(wk(a+b)), sin(wk(a+b)) COMPUTES the addition via trig identities
WL = WUWout cos(wk·c), sin(wk·c) directions READS OFF the answer by shifting by -c
Softmax exp(logitc) / Σ exp(logitj) SELECTS the c with highest constructive interference
The sine is not "generated" at runtime from scratch. It is baked into the weights during training. The embedding matrix literally IS a lookup table of sines and cosines. The rest of the network is wired to combine them using trig identities. The whole thing is a learned Fourier calculator!
